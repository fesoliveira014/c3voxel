# M8 — Deferred Lighting Chain — Implementation Plan

**Spec:** `docs/superpowers/specs/2026-04-20-m8-deferred-lighting-design.md`
**Approach:** ship in a single squash-merge-style branch; each task below is a self-contained step that leaves the codebase compiling (`c3c build linux`) and, wherever possible, runnable.

---

## Task sequence

### T1 — Volume packed output (foundation)

Change `generate.comp.glsl` so the volume texture no longer stores baked RGB color. Volume now stores:
- `.rg` = 0 (placeholder for normal; T2 fills it)
- `.b`  = `material_id / 255.0`
- `.a`  = opacity

Strip `material_color()` and the voronoi-modulation multiply. Delete the big switch.

Files: `resources/shaders/generate.comp.glsl`.

**Verify:** build passes. Running the app renders mostly-black (colors removed) — expected. Colors return in T6.

### T2 — Volume normal pass

New compute shader `resources/shaders/generate_normal.comp.glsl`. One thread per voxel. Samples `volume.a` with 6-tap central difference to compute gradient, normalizes, octahedral-packs, writes into `.rg` of the same volume via an RG8 aux texture ping-pong.

C3 glue in `src/voxel/generate.c3`: add `normal_shader` + `aux_volume_ring` fields; dispatch after the main generate, before jump_flood. `aux_volume_ring` reuses the `VolumeRing` pattern — 4 slots of RG8 128³ (16 MB total).

Add `resources/shaders/common/octahedral.glsl` (helper include) with `octahedral_pack/unpack`.

Files:
- new: `resources/shaders/generate_normal.comp.glsl`, `resources/shaders/common/octahedral.glsl`
- edit: `src/voxel/generate.c3`, `src/voxel/volume.c3` (aux ring)

**Verify:** build passes. Rendering still mostly-black but the volume `.rg` now contains packed normals.

### T3 — Holder FBO → 3 MRTs

Holder FBO grows from 2 to 3 color attachments:
- `colors[0]` R8UI   material_id
- `colors[1]` RG8    octahedral normal
- `colors[2]` R16F   height (unchanged format, just reindexed)

Update:
- `src/world/holder.c3::HolderPool.get_or_create` — add R8UI + RG8 attachments.
- `resources/shaders/raymarch.comp.glsl` — output MRTs via three `imageStore` calls. On miss, write `material=0, normal=(0.5,0.5), height=-1e30`.
- `src/voxel/raymarch.c3::dispatch_holder` — bind three output images instead of two.

Files: `src/world/holder.c3`, `src/voxel/raymarch.c3`, `resources/shaders/raymarch.comp.glsl`.

**Verify:** build passes. The existing composite.frag breaks (wrong bindings) — will be fixed in T4.

### T4 — Composite → screen G-buffer (render::deferred)

Rewrite today's `src/render/composite.c3` + `resources/shaders/composite.frag.glsl` as `src/render/deferred.c3` + `resources/shaders/deferred.frag.glsl`. Output becomes the single-texture screen G-buffer (RGBA8):

```
.r, .g   octahedral normal   (passed through from winning holder)
.b       height / MAX_WORLD_Y
.a       material_id / 255
```

Still picks the max-height holder per pixel; just writes the packed format instead of baked color. Allocate the G-buffer texture at the window framebuffer size.

Files:
- new: `src/render/deferred.c3`, `resources/shaders/deferred.frag.glsl`
- delete: `src/render/composite.c3`, `resources/shaders/composite.{vert,frag}.glsl`
- edit: `src/main.c3` (frame loop references the new module)

**Verify:** build passes. Running shows a grayscale-ish shape since material/normal/height are packed but not yet colorized — expected until T6.

### T5 — Palette LUT + material table

Expose `MATERIAL_COLORS[17]` as a constant in `src/voxel/material.c3`, replacing the GLSL switch's data (walk it off the existing palette in `generate.comp.glsl`). Upload to a `gpu::Texture2D` 17×1 RGBA8 at init in `src/render/lighting.c3`.

Files:
- edit: `src/voxel/material.c3`
- new: `src/render/lighting.c3` (stub; T6 adds the shader)

**Verify:** build passes.

### T6 — Lighting pass (no shadow yet)

`src/render/lighting.c3` + `resources/shaders/lighting.frag.glsl`:
- Reads the screen G-buffer and palette LUT.
- Computes `lit = palette[material] * (ambient + sun_color * NdotL)`.
- No AO yet (bind a 1×1 white texture as `ao_tex`).
- No shadow yet (hardcode `shadow = 1.0`).
- Writes to a result RGBA8 FBO.

Also add the minimal `render::present` module: `src/render/present.c3` + `resources/shaders/present.frag.glsl` (gamma 1/2.2, blit to default framebuffer).

Wire both into `src/main.c3` frame loop.

Files:
- edit: `src/render/lighting.c3`
- new: `resources/shaders/lighting.frag.glsl`, `src/render/present.c3`, `resources/shaders/present.frag.glsl`
- edit: `src/main.c3`

**Verify:** colored-and-lit scene renders, shadows and AO absent.

### T7 — time_of_day module + sun dir/color

`src/game/time_of_day.c3`:
- Struct `TimeOfDay { float t; bool auto_advance; core::Seconds day_length; float key_step; }`.
- Methods: `update(dt)`, `step(dir)`, `toggle_auto()`.
- Pure functions: `sun_direction_for(t)`, `sun_color_for(t)` (returns `Vec4` with RGB + intensity in `.a`).

Wire into `src/main.c3`:
- Instantiate, update each frame.
- Keys `[` / `]` call `.step(∓1)` and disable auto.
- Key `P` calls `.toggle_auto()`.
- Fill `LightingUniforms.sun_dir` and `.sun_color` from the functions each frame.

Tests in `test/game/time_of_day_test.c3`:
- Assert known direction values at `t = 0, 0.25, 0.5, 0.75`.
- Assert brightness monotonic through day arc.

Files:
- new: `src/game/time_of_day.c3`, `test/game/time_of_day_test.c3`
- edit: `src/main.c3`, `src/render/lighting.c3` (consume uniforms)

**Verify:** `c3c compile-test` passes; scene shows directional shading that changes over time.

### T8 — Shadow march

Add `resources/shaders/common/iso.glsl` with `iso_forward(world) → vec2 uv` and `iso_inverse(uv, height) → vec3 world`. Mirror the y-up sy convention from `src/game/camera.c3`.

Add the `march_shadow` function inside `lighting.frag` per spec §6: simultaneous world+screen `mix` for 24 steps, +2.0 bias, water/glass/air filter. Hook into the lit composite.

Roundtrip test `test/core/iso_roundtrip_test.c3`: random `(world, height)` through forward/inverse matches to within 0.01.

Files:
- new: `resources/shaders/common/iso.glsl`, `test/core/iso_roundtrip_test.c3`
- edit: `resources/shaders/lighting.frag.glsl`

**Verify:** building and tree shadows visible on ground.

### T9 — AO pass

Port `docs/ssao.md` §3 shader verbatim:
- New module `src/render/ao.c3`.
- New shader `resources/shaders/ao.frag.glsl` with `I_MAX = 8`, `J_MAX = 5` (24 effective taps).
- (Optional: separate `ao_high.frag.glsl` with `I_MAX = 32`; toggle via `F4`. Can land as follow-up.)
- Dispatched between the G-buffer build and the lighting pass; writes R8 AO.

Change `lighting.frag` to sample the AO texture instead of a constant.

Files:
- new: `src/render/ao.c3`, `resources/shaders/ao.frag.glsl`
- edit: `resources/shaders/lighting.frag.glsl`, `src/main.c3`

**Verify:** corners (wall-ground joints) visibly darker; AO value in `.a` of dark_color branch.

### T10 — Composite smoothstep + polish

Swap the lighting composite from simple `albedo * (ambient + diffuse) * ao` to the `smoothstep`-gated lerp from spec §7 / lighting.md §5:

```glsl
vec3  dark_color = albedo * ambient_color * ao;
vec3  lit_color  = albedo * (light_contrib + ambient_color);
float strength   = smoothstep(0.05, 0.6, dot(light_contrib, vec3(1.0/3.0)));
out_color = vec4(mix(dark_color, lit_color, strength), 1.0);
```

Files: `resources/shaders/lighting.frag.glsl`.

**Verify:** shadow falloff looks softer, night side retains AO-modulated base color.

### T11 — Final verification and commit

- `c3c build linux` clean.
- `c3c compile-test` clean.
- Manual acceptance (from spec §11):
  1. Screenshot at `t = 0.2` vs `t = 0.4`; shadow direction differs.
  2. Building wall casts shadow on adjacent ground.
  3. Tree casts shadow on neighbouring wall.
  4. Wall-ground corners darker than wall mid-height.
  5. No page-boundary seams on flat ground.
- Commit + push each feature task individually; final commit references spec+plan.

## Risk register

| Risk | Mitigation |
|------|------------|
| Octahedral normal precision artifacts on near-axis normals | 8-bit is ~2° max error; acceptable. If visible banding appears, fall back to RG16 (+2 MB per holder). |
| Shadow self-acne despite +2 bias | `shadow_step` / bias exposed via UBO — tune without rebuild. |
| Screen G-buffer 1080p memory | 8 MB; fits comfortably. |
| Normal pass ping-pong barrier cost | One extra `SHADER_IMAGE_ACCESS` barrier per page regen; measured in `timer.dump()`. If > 0.5 ms, inline the gradient into `generate.comp.glsl` with a shared-memory pre-fetch. |
| Iso-inverse mismatch between GLSL and C3 | `test/core/iso_roundtrip_test.c3` catches it. |
| Composite smoothstep thresholds feel off | `0.05` / `0.6` are initial values; tune after T11 screenshots. |
