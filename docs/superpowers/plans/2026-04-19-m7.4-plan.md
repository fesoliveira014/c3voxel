## Task Breakdown

- `format_channel` → `GL_RED_INTEGER`.
- `format_pixel_type` → `GL_UNSIGNED_SHORT`.
- Verify `texture2d_create` picks GL_NEAREST for R16UI (already does for all UI formats via the same branch as R8UI).
- **Commit:** `m7.4: add R16UI format plumbing`

### T1 — BuildingNode + BuildingCon structs + lifecycle

- New `src/world/building.c3` with:
  - Constants (`BUILDING_TILE_PX`, `BUILDING_GRID_X/Z/Y`, `BUILDING_NODE_CONS`, `BUILDING_HEIGHT_ITERS`, `BLOCK_SIZE_IN_LOTS`, `LOT_PX`, `WALL_RAD_M`, `BUILDING_NEW_SEA_LEV`).
  - Enums: `ConType`, `ConClass`.
  - Table: `CON_TYPE_CLASSES`.
  - Bitstruct: `ConFlags : uint` — if c3c 0.7.11 treats `bitstruct : uint` as 4-B under `@packed`, proceed. Else fallback to `uint flags` + helper macros. **Test this by writing the struct + `$assert(BuildingCon.sizeof == 24)` and building**.
  - Structs: `BuildingCon`, `BuildingNode`, `MapNode`.
  - `$assert(BuildingCon.sizeof == 24)`, `$assert(BuildingNode.sizeof == 312)`, `$assert(MapNode.sizeof == 8)`.
  - Stubs: `building_data_init(&Block, Allocator) -> void?`, `building_data_free(&Block, Allocator)`, `building_data_clear(&Block)`.
- Extend `Block` (`src/world/block.c3`): add `BuildingNode[] building_data`, `MapNode[] map_data`, `bool buildings_dirty`, `gpu::Texture2D building_interior_tex`.
- Extend `BlockPool.get_or_create`: allocate the new fields (via `building_data_init`), create `building_interior_tex` as R16UI. On alloc failure, unwind already-allocated heightmap_cpu/road and return BLOCK_INVALID.
- Extend `BlockPool.destroy` / `decrement_ref`: call `building_data_free`, destroy `building_interior_tex`.
- **Commit:** `m7.4: BuildingNode/Con/MapNode + Block lifecycle`

### T2 — Main-road overlay in road.c3

- In `src/world/road.c3::generate_roads_for_block`, BEFORE `generate_maze`, if this block is a city:
  - Draw horizontal road lines at `z = 0, BLOCK_SIZE_IN_LOTS*BUILDING_TILE_PX, 2*...` across the full 512-px raster.
  - Draw vertical road lines at `x = 0, step, 2*step, ...`.
  - Use `draw_segment` with `ROAD_BIT`.
- The maze then OR-overlays its own roads on top; no logic conflict.
- Add constant `const int BUILDING_TILE_PX = 8; const int BLOCK_SIZE_IN_LOTS = 8;` in road.c3 OR import from building.c3 once it exists.
- **Commit:** `m7.4: main-road overlay in road generator`

### T3 — Build `map_data` + heightmap smoothing (phases 1 + 3)

- In `building.c3`:
  - `build_map_data(Block*)` — walk 64×64 tiles, sample road_raster at tile centre → connection_props = ROAD if set. Else sample heightmap_cpu, set `adjusted_height = floor(h_meters * MAX_HEIGHT / BUILDING_TILE_PX)`.
  - `smooth_heights(Block*)` — up to BUILDING_HEIGHT_ITERS passes, clamp each tile to `max(neighbours) - 1` and enforce `max(adjusted_height, BUILDING_NEW_SEA_LEV)`. Stop early when no changes.
- **Commit:** `m7.4: map_data build + height smoothing`

### T4 — Building selection (phase 2) + ROADS_AND_BUILDINGS (phase 4)

- `select_buildings(Block*, uint world_seed)` — for each lot (8×8 tile region), hash-driven pick of 1–4 building footprints. Mark non-road tiles as MAINHALL/WING per spec §Phase 2.5.
- `connect_nodes(Block*, i1,j1,k1, i2,j2,k2, ConType, int direction, int height_delta, int axis_layer, float wall_rad, ConFlags flags)` — writes slots `axis_layer*6 + base_dir` at endpoint 1, `axis_layer*6 + (base_dir ^ 1)` at endpoint 2. Both receive the same `con_type`, `wall_rad`, `height_delta`; endpoint 2 gets negated `direction`.
- `phase_roads_and_buildings(Block*)` — triple-loop per VQ doc §3. For each tile (i,j,k) and horizontal direction m: if mapData.connection_props[m] is ROAD/MAINHALL/WING AND `adjusted_height == k+1`, call connect_nodes with FOUNDATION type.
- **Commit:** `m7.4: building selection + roads_and_buildings phase`

### T5 — WING_TIPS + DOORS (phases 5 + 6)

- `phase_wing_tips(Block*)` — walk axis-aligned wing runs; set `wing_beg`/`wing_end` flags on endpoints; assign `wing_mult` linearly tapering from 0.6 at endpoints to 1.0 at midpoint per spec.
- `phase_doors(Block*)` — for each WING connection, check if the mapData tile at the other end has `connection_props` class CTC_ROAD. If yes → DOORWAY. Else → WINDOWFRAME; emit LANTERN on interior-side (direction == +1).
- **Commit:** `m7.4: wing_tips + doors phases`

### T6 — FILL_TERRAIN (phase 7) + interior texture upload

- `phase_fill_terrain(Block*)` — top-down walk. For each (i, j) tile column, if any node at (i, j, k) is part of a building (WING/WINDOWFRAME/DOORWAY/MAINHALL), mark bits `[adjusted_height+1 .. adjusted_height+wall_height]` in `block.interior_mask[j*512+i]` (ushort[]).
- Upload to `building_interior_tex` via R16UI upload.
- **Commit:** `m7.4: fill_terrain + interior texture upload`

### T7 — `buildings_generate_for_block` + drain_dirty wiring

- Public entry `buildings_generate_for_block(Block*, uint world_seed)` runs phases 1 → 2 → 2.5 → 3 → 4 → 5 → 6 → 7 in order.
- Call from `src/main.c3::drain_dirty`, inside `if (block.heightmap_dirty)`, after `generate_roads_for_block`.
- **Commit:** `m7.4: generate_for_block entrypoint + drain_dirty wiring`

### T8 — Geometry emission + shader interior carve

- `building_emit_to_page(PageGeometry* out, Block* block, Page* page)` walks tiles in page bounds; for each non-NONE connection emit:
  - WING/FOUNDATION → `GeomEntry` (superellipsoid, power_vals.x=6) with material PLASTER/STONE.
  - DOORWAY → GeomEntry with WOOD.
  - WINDOWFRAME → GeomEntry with GLASS.
  - LANTERN → `LineEntry` with METAL, rad0=0.3, rad1=0.5.
  - Top-level wall (k+1 == max_height) → SHINGLE material override.
- Call from `src/world/geometry.c3::geometry_generate_page` after `plant_generate_page`.
- `GenerateContext.dispatch_page` binds `block.building_interior_tex` at binding 8.
- `resources/shaders/generate.comp.glsl`:
  - `layout(binding = 8) uniform usampler2D building_interior`
  - After terrain + road + tree evaluation, sample `interior_flags = texture(building_interior, hm_uv).r`; if `interior_flags & (1u << int(world.y / BUILDING_TILE_PX))`, set `result = vec4(0)`.
- **Commit:** `m7.4: geometry emission + shader interior carve`
