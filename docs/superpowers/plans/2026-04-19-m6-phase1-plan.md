# Implementation Plan — M6 Phase 1

Spec: `docs/superpowers/specs/2026-04-19-m6-phase1-multi-page-hierarchy-design.md` (rev @ 69e69ba).

Execution: subagent-driven development. Five sequential tasks, each
dispatched to an isolated subagent. Each task produces a clean
`c3c build linux` + smoke run, then commits.

## Task 1 — Core scaffolding + coord extensions + noise

**Files:**
- NEW `src/core/hash.c3` — `alias Hash{K, V} = std::collections::map::HashMap{K, V};`. If the generic alias fails to compile, fallback to `struct Ivec3IntHash { inline std::collections::map::HashMap{IVec3, int} inner; }` wrapper (spec §15 fallback).
- NEW `src/core/pool.c3` — `alias Arena = std::core::mem::allocators::ArenaAllocator;`.
- NEW `src/core/noise.c3` — public `simplex2(float x, float z, uint seed) -> float` returning `[-1, 1]` + `fbm2(float x, float z, int octaves, float persistence, float lacunarity, uint seed) -> float`. Implementation follows `docs/noise-reference.md` §Simplex + §fBm. 3D/4D and Perlin stub to `$assert(false, "not implemented in M6 Phase 1")`.
- MODIFY `src/world/coords.c3` — add `const int ACTIVE_HOLDERS_PER_AXIS = 2;`. Leave existing constants alone.

**Build gate:** `c3c build linux` clean. No main.c3 changes, binary unchanged.

**Commit:** `M6 P1 T1: core::hash / core::pool / core::noise + world::coords extension`.

## Task 2 — World hierarchy pools

**Files:**
- NEW `src/world/page.c3` — `Page { PageCoord px, pz; Vec4 world_min, world_max; }` + `PageId = inline core::Handle` + `PagePool { Page[] storage; Hash{IVec3, int} lookup; Allocator allocator; }` with `init/destroy/get_or_create/get` methods.
- NEW `src/world/holder.c3` — `Holder { HolderCoord hx, hz; PageId[PAGES_PER_HOLDER*PAGES_PER_HOLDER] pages; gpu::Framebuffer fbo_layer0; }` + `HolderId = inline core::Handle` + `HolderPool`.
- NEW `src/world/block.c3` — `Block { BlockCoord cx, cz; HolderId[HOLDERS_PER_BLOCK*HOLDERS_PER_BLOCK] holders; gpu::Texture2D heightmap; bool heightmap_dirty; }` + `BlockId = inline core::Handle` + `BlockPool`.
- NEW `src/world/terrain.c3` — `fn void generate_heightmap_cpu(Block* block, uint seed)` that calls `core::noise::fbm2(...)` for each texel of a 512² staging buffer and uploads via `gpu::texture2d_upload`. Height values in `[0, 1]`.

**Build gate:** `c3c build linux` clean. Smoke: construct one of each pool in a throwaway test, round-trip handles. Main unchanged.

**Commit:** `M6 P1 T2: world hierarchy (Block + Holder + Page + terrain)`.

## Task 3 — VolumeRing + generate/raymarch extensions

**Files:**
- MODIFY `src/voxel/volume.c3` — add `struct VolumeRing { gpu::Texture3D[4] volumes; int current; }` with `.init`, `.destroy`, `.next() -> int` returning the next slot index (round-robin). Keep existing `struct Volume` for now (aliases to `VolumeRing` slot-0 semantics until main rewires) or mark deprecated.
- MODIFY `src/voxel/generate.c3`:
  - Add `gpu::Texture2D heightmap` binding (unit 1) to dispatch.
  - UBO `GenerateUniforms` gains `Vec4 block_origin` + `Vec4 block_extent` fields.
  - Signature: `GenerateContext.dispatch(&self, Commands*, VolumeRing*, int slot, world::Page* page, gpu::Texture2D block_heightmap, Seconds time)`.
- MODIFY `resources/shaders/generate.comp.glsl`:
  - Add `layout(binding = 1) uniform sampler2D heightmap;`.
  - Per-voxel: sample heightmap at `(world.xz - block_origin.xz) / block_extent.xz`, compare to `world.y`, produce band colors (stone/dirt/grass) when `world.y <= h`, else empty.
  - Keep existing time-based hash carve as subordinate variation.
- MODIFY `src/voxel/raymarch.c3`:
  - `RayMarchUniforms` gains `IVec4 write_offset` (x, y = pixel offset into output FBO; z, w reserved).
  - Field `page_pixels : IVec4` (x, y = sub-rect size in pixels = `PIXELS_PER_PAGE` per axis).
  - New entrypoint `RayMarchContext.dispatch_into(&self, Commands*, VolumeRing*, int slot, DistanceField*, world::Page* page, gpu::Framebuffer* target, IVec2 write_offset)`.
  - `target` is the Holder's `fbo_layer0`; bind color attachment as image unit 2 (rgba8), height attachment as image unit 3 (r16f). The existing `.dispatch(...)` to a standalone 2D texture can remain for back-compat tests.
- MODIFY `resources/shaders/raymarch.comp.glsl`:
  - Two MRT-style outputs: `layout(binding=2, rgba8) writeonly image2D color_out;` and `layout(binding=3, r16f) writeonly image2D height_out;`.
  - NDC and ray derive from `gl_GlobalInvocationID.xy / vec2(page_pixels.xy)`; only the `imageStore` position uses `gl_GlobalInvocationID.xy + write_offset.xy`.
  - On hit: write color to `color_out`, write world-y to `height_out`. On miss: write transparent + very-negative height.

**Build gate:** `c3c build linux` clean. Hot-reload of the modified shaders works. Smoke: temporary test in main dispatches one page into a 512² test framebuffer and confirms pixels written.

**Commit:** `M6 P1 T3: VolumeRing + generate heightmap + raymarch MRT + sub-rect writes`.

## Task 4 — Composite pass

**Files:**
- NEW `src/render/composite.c3`:
  ```c3
  module c3voxel::render;

  struct CompositeUniforms @packed @align(16) {
      Vec4[4] screen_min;
      Vec4[4] screen_max;
      IVec4 screen_size;
      IVec4 active_count;
  }

  struct CompositeContext { gpu::Shader shader; gpu::Buffer ubo; }

  fn void? CompositeContext.init(&self, Allocator allocator);
  fn void CompositeContext.destroy(&self);
  fn void CompositeContext.dispatch(&self, gpu::Commands* cmd, world::Holder*[4] holders, game::CameraState* cam, Pixel fb_w, Pixel fb_h);
  ```
- NEW `resources/shaders/composite.vert.glsl`:
  Fullscreen triangle using `gl_VertexID`, emits `v_pixel = gl_Position.xy * 0.5 + 0.5` scaled by screen size.
- NEW `resources/shaders/composite.frag.glsl`:
  4 pairs of hard-coded samplers (`holder0_color`, `holder0_height`…`holder3_color`, `holder3_height`) at bindings 0..7. Unrolled 4-iteration max-height compare. UBO at binding 8 with the per-holder screen rects + active count. On miss → discard.

**Build gate:** `c3c build linux` clean. Standalone test: clear screen, dispatch composite with 4 dummy single-color textures → observe expected overlap.

**Commit:** `M6 P1 T4: composite pass (MRT max-height → screen)`.

## Task 5 — Main orchestration + smoke

**Files:**
- MODIFY `src/main.c3`:
  - Allocate `BlockPool`, `HolderPool`, `PagePool` with `mem`.
  - Create 1 Block via `block_pool.get_or_create(0, 0)`. CPU heightmap via `world::generate_heightmap_cpu(block, seed=1)`.
  - Create 4 Holders via `holder_pool.get_or_create(hx, hz)` for `hx, hz ∈ [0, 2)`. Each holder's `fbo_layer0` is a 512² `RGBA8+R16F` MRT.
  - Create 16 Pages and wire into Holders (4 pages per holder).
  - `VolumeRing` with 4 volumes; existing `DistanceField` ping-pong extended to 4 pairs (or iterated per-page with single DF reused, whichever is simpler — spec doesn't dictate).
  - Startup loop: for each of 16 pages → volume_ring.next() → gen.dispatch → jfa.build → rm.dispatch_into(holder.fbo, write_offset).
  - Per-frame: `composite.dispatch(cmd, holders, &cam, win.fb_width, win.fb_height)`.
  - Keep hot-reload covering all shaders (fullscreen vs. composite, generate, raymarch, distance_field init/step).
  - Keep FPS print.

**Build gate:** `c3c build linux` clean. Smoke run: binary starts, terrain grid visible, no GL errors, ≤ 16 ms/frame at 1080p on RTX 4090.

**Commit:** `M6 P1 T5: main wiring — 4×4 terrain grid rendering`.

## Execution

Dispatch subagents one at a time in the order above. Each subagent:

1. Reads this plan + the spec + relevant existing files.
2. Implements only its assigned task.
3. Runs `c3c build linux` and a short smoke (no regression vs prior commit).
4. Commits with the prescribed message.
5. Returns a short summary.

Main agent validates commit SHA + build status after each task, then dispatches the next.
