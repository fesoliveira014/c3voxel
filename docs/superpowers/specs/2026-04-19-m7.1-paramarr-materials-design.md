# M7.1 — `paramArr` Pipeline + Voronoi + Materials

**Status:** Spec
**Date:** 2026-04-19
**Predecessor:** M6.5 (closed)
**Successors:** M7.2 (Trees), M7.3 (Roads), M7.4 (Buildings)
**Reference:** `docs/procedural-generation-vq.md` §5 (Voronoi), §6 (Materials + paramArr layout)

## Goal

Ship the procedural-geometry infrastructure that all of M7's later phases build on: an SSBO-backed per-page geometry description, a shared GPU evaluator, the full VoxelQuest material ID table, and a CPU-generated voronoi seed set. No real geometry yet — M7.1 ships a 3×3 smoke-test grid of superellipsoids per page so the pipeline is exercised end-to-end before trees and buildings start producing hundreds of entries.

## Scope

### In

- SSBO-backed `paramArr` pipeline, per-page, grow-as-needed (cap `MAX_ENTRIES_PER_PAGE = 1024`).
- Three typed entry SSBOs: `GeomEntry`, `TreeEntry`, `LineEntry`. The shader iterates all three, contributing to one shared nearest-match tracker so cross-type corner detection still works.
- Full VQ material ID enum (17 entries including `NULL`).
- CPU-generated voronoi seeds — 27 per page, deterministic hash of page coord, uploaded via UBO alongside paramArr.
- `generate.comp.glsl` rewritten: per voxel, iterate the 3 typed arrays, evaluate the closest entry's shape (superellipsoid/Bézier/line), sample voronoi for material variation, write RGBA + material-ID tag.
- Smoke test: 9 hardcoded superellipsoids per page (3×3 grid, varying materials).

### Out (owned by later phases)

- **M7.2:** Tree generation (L-system + Bézier geometry into `TreeEntry[]`).
- **M7.3:** Road generation (midpoint displacement + maze). Writes a "road pixel" channel that M7.4 reads.
- **M7.4:** Building grammar (phased generation on 3D node grid → `GeomEntry[]` + `LineEntry[]` for lanterns).
- Heightmap stitching (multi-image blending via simplex weights). Current `fbm2` terrain stays.
- Lighting, shadows, palette LUT (M8).

### `src/voxel/geometry.c3` (new)

`GeomEntry` mirrors VoxelQuest's `E_GP_*` slot layout (doc §6.2), shifted into a typed C3 struct. `TreeEntry` and `LineEntry` mirror `E_TP_*` and `E_AP_*` respectively; they are defined now as empty-but-size-stable shells so M7.2's SSBO layouts don't churn.

```c3
module c3voxel::voxel;

import c3voxel::core;

struct MaterialParams @packed {
    float material_id;
    float normal_id;
    float modifier;
    float _pad;
}
$assert(MaterialParams.sizeof == 16);

// Superellipsoid/bounded-shape entry. Mirrors VQ's E_GP_* layout.
// std430 layout — since all fields are Vec4 / 16-B struct / float[N], std430
// packs identically to std140 here.
//
// Fields not read by the M7.1 smoke evaluator (corner_dis, power_vals_2,
// thick_vals) are **reserved to mirror VQ's layout** so M7.2+ consumers can
// fill them in without a struct churn. The smoke generator zeroes them.
struct GeomEntry @packed @align(16) {
    core::Vec4  vis_min;          // visibility AABB (skip entry if voxel outside)
    core::Vec4  vis_max;
    core::Vec4  bounds_min;        // superellipsoid fit AABB
    core::Vec4  bounds_max;
    core::Vec4  corner_dis;         // reserved (VQ E_GP_CORNERDISINPIXELS)
    core::Vec4  power_vals;         // superellipsoid exponents p (x), q (y)
    core::Vec4  power_vals_2;        // reserved
    core::Vec4  thick_vals;          // reserved
    core::Vec4  center_point;
    MaterialParams mat;              // 16 B
}
$assert(GeomEntry.sizeof % 16 == 0);  // 160 B = 10 × 16

// Bezier-branch entry. Geometry is P0/P1/P2 with begin/end thickness + leaf sphere.
// Fields defined now so M7.2's Bezier evaluator has a stable struct. Not yet emitted.
struct TreeEntry @packed @align(16) {
    core::Vec4 vis_min;
    core::Vec4 vis_max;
    core::Vec4 p0;
    core::Vec4 p1;
    core::Vec4 p2;
    core::Vec4 power_vals;
    core::Vec4 power_vals_2;
    core::Vec4 thick_vals;          // begThickness (x), endThickness (y), sphereRad (z)
    core::Vec4 center_point;
    MaterialParams mat;
}
$assert(TreeEntry.sizeof % 16 == 0);

// Line-geometry entry (lanterns, thin walls). Matches VQ's E_AP_*.
struct LineEntry @packed @align(16) {
    core::Vec4 vis_min;
    core::Vec4 vis_max;
    core::Vec4 origin;
    core::Vec4 tangent;
    core::Vec4 bitangent;
    core::Vec4 normal;
    core::Vec4 rad0;                // rad0 (x), rad1 (y)
    core::Vec4 rad1;
    core::Vec4 center_point;
    MaterialParams mat;
}
$assert(LineEntry.sizeof % 16 == 0);
```

**Size invariant:** each entry is a multiple of 16 B (std430 + `@align(16)` + only 16-B members). Sizes are **not** required to match across types — each has its own GLSL array with its own stride. Today all three land on 160 B; future phases may grow any of them independently.

**SSBO sizing:** `MAX_ENTRIES_PER_PAGE * sizeof(T)` per array (e.g. `1024 * 160 = 160 KB` at current sizes). Re-asserted at spec-update time if a type grows.

### `src/voxel/voronoi.c3` (new)

```c3
module c3voxel::voxel;

import c3voxel::core;
import c3voxel::world;

const int VORONOI_SEEDS_PER_PAGE = 27;   // 3×3×3 grid

struct VoronoiParams @packed @align(16) {
    core::Vec4[VORONOI_SEEDS_PER_PAGE] seeds;  // xyz = world pos; w = seed id
}
$assert(VoronoiParams.sizeof % 16 == 0);

// Deterministic seed generation: hash(page coord + cell index) → perturbed grid center.
fn VoronoiParams voronoi_for_page(world::Page* page);

// Internal: hash used for both position perturbation and per-cell ID.
fn uint  voronoi_hash(uint x, uint y, uint z, uint cell) @local;
fn float voronoi_frand(uint h) @local;        // [0, 1)
```

Perturbation magnitude: ±0.4 of a cell's third-of-page extent (i.e., seeds never stray far enough to collide with another cell's neighborhood). Cell centers are at `(⅙, ½, ⅚)` fractions along each axis of the page AABB.

### `src/world/geometry.c3` (new)

Per-page geometry storage + the CPU-side generator.

```c3
module c3voxel::world;

import c3voxel::voxel;

struct PageGeometry {
    List{voxel::GeomEntry} geom;
    List{voxel::TreeEntry} tree;
    List{voxel::LineEntry} line;
    voxel::VoronoiParams    voronoi;
}

fn void PageGeometry.init(&self, Allocator allocator);
fn void PageGeometry.free(&self);
fn void PageGeometry.clear(&self);              // keep capacity, reset length; reused on page-slot recycle

// M7.1 smoke-test generator: 3×3 grid of superellipsoids centered on the page
// surface (Y = heightmap(center)+epsilon), sized ~1/4 of a cell, materials
// cycling through STONE, BRICK, PLASTER, WOOD, SHINGLE, MORTAR, DIRT, GRASS, SAND.
// Reserved GeomEntry fields (corner_dis, power_vals_2, thick_vals) are zeroed.
fn void geometry_generate_smoke_test(&PageGeometry out, world::Page* page);
```

`Page` struct gains:
```c3
struct Page {
    // ... existing fields ...
    PageGeometry geometry;    // owned; List{T} heap-allocated through the PagePool allocator
}
```

**Lifecycle:** `PageGeometry.init(pool.allocator)` is called when a new `Page` is created in `PagePool.get_or_create`. `PageGeometry.clear()` is called on page-slot recycle (same point where the old handle is bumped). `PageGeometry.free()` is called from `PagePool.destroy`. No new arena — we reuse the `PagePool` allocator that already owns the pool's pages, matching the pattern used by `DirtyQueue` (`List{T}` on a caller-supplied `Allocator`). Spec's earlier mention of "arena" was stylistic; the concrete model is `List{T}` on the pool allocator.

## GPU Pipeline

### SSBO bindings (generate.comp.glsl)

Renumber existing bindings to make room for three SSBOs + voronoi UBO:

| Binding | Name           | Type                  | Access     |
|---------|----------------|-----------------------|------------|
| 0       | `volume`       | `image3D rgba8`       | writeonly  |
| 1       | `heightmap`    | `sampler2D`           | sampled    |
| 2       | `U`            | UBO (existing)        | read       |
| 3       | `GeomEntries`  | SSBO `GeomEntry[]`    | readonly   |
| 4       | `TreeEntries`  | SSBO `TreeEntry[]`    | readonly   |
| 5       | `LineEntries`  | SSBO `LineEntry[]`    | readonly   |
| 6       | `VoronoiU`     | UBO                    | read       |

`GenerateUniforms` gains a trailing `IVec4 entry_counts` (`x=num_geom`, `y=num_tree`, `z=num_line`, `w=0`). Current struct is 80 B; after the addition it is 96 B. The existing `$assert(GenerateUniforms.sizeof % 16 == 0)` continues to hold. Updated layout:

```c3
struct GenerateUniforms @packed @align(16) {
    core::Vec4    world_min;
    core::Vec4    world_max;
    core::Vec4    block_origin;
    core::Vec4    block_extent;
    core::Seconds time;
    float[3]      _pad;
    core::IVec4   entry_counts;    // num_geom, num_tree, num_line, reserved
}
$assert(GenerateUniforms.sizeof % 16 == 0);
```

Note on the "5th UBO" comment in `generate.c3`: M7.1 adds `VoronoiParams` as the fifth UBO in the project. Per the promotion note in `docs/ubo-convention.md`, that would trigger the `@std140` macro work — but M7.1 is the first phase to actually add a UBO in the "uniform geometry upload" category, and the macro promotion is not worth scope expansion here. The doc note is extended to "6th UBO" instead.

### Shader flow (high-level)

```glsl
void main() {
    ivec3 voxel = ivec3(gl_GlobalInvocationID);
    vec3  world = // compute as today

    // 1. Terrain base (existing) — used as fallback color.
    float h01   = texture(heightmap, hm_uv).r;
    float h_w   = h01 * MAX_HEIGHT;
    vec4  terrain = terrain_color_band(h01, world.y, h_w);

    // 2. Nearest-match across all three typed arrays with shared tracker.
    //
    // VQ's original (doc §6.3) wraps this in an outer loop over MATERIAL IDs
    // ("for each material pass, find the closest entry with matParams.x == curMat").
    // We intentionally collapse that into a single pass: the smoke test has one
    // entry per cell so vis_min/vis_max culling alone selects a winner. If
    // material-pass composition (e.g. mortar between bricks) becomes needed,
    // reintroduce the outer loop — the data shape already supports it.
    float best_dis      = 1e30;
    float next_best_dis = 1e30;
    int   best_idx  = -1;
    int   best_type = -1;   // 0 = geom, 1 = tree, 2 = line

    // geom pass
    for (int i = 0; i < entry_counts.x; i++) {
        GeomEntry e = geom[i];
        if (!aabb_contains(world, e.vis_min.xyz, e.vis_max.xyz)) continue;
        float d = distance(world, e.center_point.xyz);
        if (d < best_dis) {
            next_best_dis = best_dis; best_dis = d;
            best_idx = i; best_type = 0;
        } else if (d < next_best_dis) {
            next_best_dis = d;
        }
    }
    // tree pass (symmetric, M7.2 populates it)
    for (int i = 0; i < entry_counts.y; i++) { /* same shape */ }
    // line pass (symmetric, M7.2+ populates it)
    for (int i = 0; i < entry_counts.z; i++) { /* same shape */ }

    // 3. Evaluate closest entry's shape.
    vec4 result = terrain;
    if (best_type == 0) {
        result = eval_geom_superellipsoid(world, geom[best_idx]);
    } else if (best_type == 1) {
        result = eval_tree_bezier(world, tree[best_idx]);   // M7.2
    } else if (best_type == 2) {
        result = eval_line(world, line[best_idx]);           // M7.2+
    }

    // 4. Voronoi material variation (applied uniformly once an entry hits).
    if (best_type >= 0) {
        // voronoi_lookup returns `{ voro_grad, voro_id, bestPos }` — see doc §5.
        // Only voro_grad is used here; voro_id is reserved for M7.4's per-cell
        // color variation. Shape is GLSL-local; not a C3 type.
        float voro_grad = voronoi_lookup_grad(world);
        result.rgb *= mix(0.85, 1.00, voro_grad);            // gentle per-cell dim
    }

    imageStore(volume, voxel, result);
}
```

### Superellipsoid evaluation

Direct port of VQ `GenerateVolume.c` (doc §6.5):

```glsl
vec4 eval_geom_superellipsoid(vec3 world, GeomEntry e) {
    vec3  center = e.center_point.xyz;
    vec3  abs_d  = abs(world - center);
    float p      = e.power_vals.x;
    float q      = e.power_vals.y;

    vec3 coef_min = (e.bounds_max.xyz - e.bounds_min.xyz) * 0.5;
    vec3 coef_max = coef_min * 1.05;   // thin ramp band for AA later

    vec3 td_min = abs_d / coef_min;
    vec3 td_max = abs_d / coef_max;
    float r_min = 1.0 / pow(pow(td_min.x, p) + pow(td_min.y, p), 1.0/p);
    float r_max = 1.0 / pow(pow(td_max.x, p) + pow(td_max.y, p), 1.0/p);
    // q for Z, combined
    // ... full formula per doc §6.5 ...

    float inside = clamp((1.0 - r_min) / (r_max - r_min), 0.0, 1.0);
    if (inside < 0.5) return vec4(0);
    vec3 color = material_color(int(e.mat.material_id));
    return vec4(color, 1.0);
}
```

## Per-Page Upload Flow

1. **Streaming marks holder dirty** (no change from M6.5).
2. **`drain_dirty` picks a holder**, iterates its 4 pages:
   - For each page: `geometry_generate_smoke_test(&page.geometry, page)` (CPU fill into the page's `PageGeometry.geom/tree/line` lists).
   - Upload `page.geometry.geom[..]` → SSBO, same for tree/line.
   - Upload `page.geometry.voronoi` → UBO.
   - Dispatch `generate.comp` (now consumes all three SSBOs + voronoi).
3. **Raymarch proceeds as today** — it only consumes the 3D volume; it does not look at paramArr.

**SSBO lifetime:** `GenerateContext` owns 3 persistent SSBOs + 1 voronoi UBO, each sized `MAX_ENTRIES_PER_PAGE * sizeof(T)`. Each page's entries are uploaded into a known offset (start of buffer — page dispatches are serialized by the barrier after each). The SSBOs are reused across pages; no per-page alloc.

**Allocator:** `PageGeometry` stores its entries in `List{T}` instances backed by the `PagePool.allocator` (same allocator the pool already uses). `List.free()` on page destruction releases the lists; `List.clear()` on page-slot recycle resets length while keeping capacity. No new allocator type is introduced.

## Implementation Order

1. **Material enum + debug palette** — pure addition, no compile impact elsewhere.
2. **`GeomEntry`/`TreeEntry`/`LineEntry` structs + `$assert`** — pure addition.
3. **Voronoi seed generator (CPU)** — pure addition, testable in isolation.
4. **`PageGeometry` + `geometry_generate_smoke_test`** — new module, hooks into `Page`.
5. **SSBO creation in `GenerateContext`** — new fields + init/destroy.
6. **`generate.comp.glsl` rewrite** — adds the three loops + shape evaluator.
7. **`drain_dirty` wires the geometry generator + uploads** — smallest correct change.

Each step committable on its own; 6 + 7 ship together.

## Rollback

All changes are additive up to the `generate.comp.glsl` rewrite. Pre-M7.1 shader stays in git history; rollback = revert the rewrite commit + the drain_dirty upload commit. `GeomEntry`/etc. files can stay in-tree (they become unused) without affecting runtime.
