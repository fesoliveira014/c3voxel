# M7.4 — Buildings (phased grammar on 3D node grid)

**Status:** Spec
**Date:** 2026-04-19
**Predecessor:** M7.3 (Roads, closed)
**Successor:** M8 (Deferred lighting)
**Reference:** `docs/procedural-generation-vq.md` §3 (Buildings)

## Goal

Port VoxelQuest's building generator: a 3D `BuildingNode` grid whose 12 per-node connections are filled in across 5 phases (roads_and_buildings, wing_tips, doors, fill_terrain — basements skipped as in VQ). Walls, doors, windows, and lanterns emerge from the graph as `GeomEntry` / `LineEntry` records into the per-page SSBOs M7.1 set up. Completes the M7 "Done when" bar: a village of 5–10 buildings with recognisable walls, roofs, doorways, windows.

## Scope

### In
- 3D `BuildingNode` grid per block. **8 world units per tile**. Grid is `64 × 64 × 16` per block (~65k nodes × ~310 B = ~20 MB per block; ×4 active blocks ≈ 80 MB CPU).
- `BuildingCon` struct matching VQ with 12 connections per node (6 axial dirs × 2 layers — foundation + superstructure). `ConType` enum (NONE/ROAD/MAINHALL/WING/FOUNDATION/DOORWAY/WINDOWFRAME/LANTERN).
- `MapNode[]` 2D layer (64×64) for road-adjacency + adjusted-height bookkeeping.
- **Main-road overlay** added to M7.3's `generate_roads_for_block`: straight roads at fixed intervals (`BLOCK_SIZE_IN_LOTS` = 8 tiles) form city blocks around the maze. Matches VQ's `blockSizeInLots` behaviour.
- **Full 5-phase generation** (minus basements, per VQ):
  - `E_BG_ROADS_AND_BUILDINGS`: read mapData, extrude foundations under WING tiles, wall framework above.
  - `E_BG_WING_TIPS`: propagate `BC_FLAG_WING_BEG`/`_WING_END` along axis-aligned wing runs; set `wing_mult` taper value.
  - `E_BG_DOORS`: for each WING connection, check adjacent tile for ROAD — if yes upgrade to DOORWAY, else WINDOWFRAME; emit LANTERN on interior-side WINDOWFRAME.
  - `E_BG_FILL_TERRAIN`: top-down walk marks inside-room voxels as air.
- **Height consistency iteration** (up to 16 passes) over `mapData.adjusted_height`: clamp each tile to `max(neighbours) - 1`, enforce water-level floor. Prevents one-tile cliffs between adjacent building tiles.
- Geometry emission per page: walls → superellipsoid `GeomEntry`, lanterns → `LineEntry`, interior-carve texture.
- Flat roofs: top-level wall tile gets `Material.SHINGLE`. Pitched roofs deferred to follow-up task #20.

### Out (follow-ups)
- Pitched / gabled roofs (task #20).
- Basements (commented out in VQ).
- Multi-story interiors / stairs.
- Entity-picked buildings (M10 editor).
- Building palette variation beyond material swaps.

## Data Shapes

### `src/world/building.c3` (new)

```c3
module c3voxel::world;

import std::core::mem::alloc;

import c3voxel::core;
import c3voxel::voxel;

const int BUILDING_TILE_PX       = 8;
const int BUILDING_GRID_X        = BLOCK_HEIGHTMAP_PIXELS / BUILDING_TILE_PX;   // 64
const int BUILDING_GRID_Z        = BUILDING_GRID_X;
const int BUILDING_GRID_Y        = 16;                                           // Z-levels
const int BUILDING_NODE_DIRS     = 6;
const int BUILDING_NODE_LAYERS   = 2;                                            // foundation / superstructure
const int BUILDING_NODE_CONS     = BUILDING_NODE_DIRS * BUILDING_NODE_LAYERS;    // 12
const int BUILDING_HEIGHT_ITERS  = 16;
const int BLOCK_SIZE_IN_LOTS     = 8;                                            // tiles per lot; main road every 8 tiles
const int LOT_PX                 = BLOCK_SIZE_IN_LOTS * BUILDING_TILE_PX;        // 64 world units per lot
const float WALL_RAD_M           = 0.5f;                                          // half-thickness, world units

enum ConType : char {
    NONE,
    ROAD,
    MAINHALL,
    WING,
    FOUNDATION,
    DOORWAY,
    WINDOWFRAME,
    LANTERN,
}
const int CON_TYPE_COUNT = 8;

enum ConClass : char {
    CTC_NONE,
    CTC_ROAD,
    CTC_WING,
    CTC_LIGHT,
}

// Lookup for doc §3 style ctClasses[conType]. Matches VQ taxonomy.
const ConClass[CON_TYPE_COUNT] CON_TYPE_CLASSES = {
    CTC_NONE,     // NONE
    CTC_ROAD,     // ROAD
    CTC_WING,     // MAINHALL
    CTC_WING,     // WING
    CTC_WING,     // FOUNDATION
    CTC_WING,     // DOORWAY
    CTC_WING,     // WINDOWFRAME
    CTC_LIGHT,    // LANTERN
};

bitstruct ConFlags : uint {
    bool inside      : 0;
    bool wing_beg    : 1;
    bool wing_end    : 2;
}

struct BuildingCon @packed {
    ConType  con_type;
    char[3]  _pad;
    ConFlags flags;
    float    wing_mult;
    float    wall_rad_m;
    int      height_delta;
    int      direction;          // -1, 0, +1 surface normal
}
$assert(BuildingCon.sizeof == 24);

struct BuildingNode @packed {
    BuildingCon[BUILDING_NODE_CONS] con;     // 12 × 24 = 288 B
    int   maze_index;
    int   id;
    int   visited;
    float power_u;
    float power_v;
    bool  near_terrain;
    bool  near_air;
    char[2] _pad;
}
$assert(BuildingNode.sizeof == 312);

struct MapNode @packed {
    float   adjusted_height;                  // tile-height units
    ConType[4] connection_props;              // +X, -X, +Z, -Z what type
}
$assert(MapNode.sizeof == 8);

// Lifecycle: allocated in BlockPool.get_or_create, freed in decrement_ref.
fn void building_data_init(&Block block, Allocator allocator);
fn void building_data_free(&Block block, Allocator allocator);
fn void building_data_clear(&Block block);

// Full generator: runs the 5 phases. Called after road generation in drain_dirty.
fn void buildings_generate_for_block(Block* block, uint world_seed);

// Emission step: walks block's building_data and appends overlapping
// TreeEntry/GeomEntry/LineEntry records into the page's geometry lists.
fn void building_emit_to_page(PageGeometry* out, Block* block, Page* page);
```

## Algorithms

### Phase 0 — Main-road overlay (extends M7.3)

In `src/world/road.c3::generate_roads_for_block` **before** `generate_maze`, if this block is a city:

```c3
int step = BLOCK_SIZE_IN_LOTS * BUILDING_TILE_PX;    // 64 world pixels
for (int z = 0; z < BLOCK_HEIGHTMAP_PIXELS; z += step) {
    draw_segment(&block.road, 0, z, BLOCK_HEIGHTMAP_PIXELS - 1, z, (char)ROAD_BIT);
}
for (int x = 0; x < BLOCK_HEIGHTMAP_PIXELS; x += step) {
    draw_segment(&block.road, x, 0, x, BLOCK_HEIGHTMAP_PIXELS - 1, (char)ROAD_BIT);
}
```

Creates a main-road grid inside the block. Maze fills between.

### Phase 1 — Build `map_data`

For each `(i, j)` in `BUILDING_GRID_X × BUILDING_GRID_Z`:
1. Sample `block.road.pixels` at tile-centre pixel `(i * TILE + TILE/2, j * TILE + TILE/2)`. If `ROAD_BIT` → `map.connection_props[*]` = `ROAD`.
2. Else, sample `block.heightmap_cpu` at tile centre → `map.adjusted_height = floor(height_meters / TILE_PX)`.

### Phase 2 — Building selection

For each lot (bounded by main-road tiles), iterate eligible (non-road) interior tiles. Hash-driven per lot:
- Pick a rectangular WING footprint 2×2 .. 3×3 tiles somewhere inside the lot.
- Mark those tiles' `connection_props` = `WING`.
- One tile in the centre of each footprint becomes `MAINHALL` (drives door-hall orientation).

Lot bounds derived from main-road grid: each lot is `BLOCK_SIZE_IN_LOTS × BLOCK_SIZE_IN_LOTS` tiles minus the surrounding road cells.

### Phase 3 — Height smoothing (up to 16 passes)

```
do {
    changed = false
    for each (i, j):
        m = map[i,j].adjusted_height
        p = max over 4 neighbours of map[nbr].adjusted_height
        if p - m > 1:
            map[i,j].adjusted_height = p - 1
            changed = true
        if newSeaLev > map[i,j].adjusted_height:
            map[i,j].adjusted_height = newSeaLev
            changed = true
    counter++
} while (changed && counter < BUILDING_HEIGHT_ITERS)
```

Clamps cliff deltas to 1 tile. `newSeaLev = 2` tile-units (= y=16 world). Same water-level constant road module uses.

### Phase 4 — `E_BG_ROADS_AND_BUILDINGS`

Full triple loop (`i, j, k, m`) per VQ doc §3 "Phase detail":

```
for each tile (i, j, k):
    for each horizontal direction m in 0..3:
        curBT = mapData[i,j].connection_props[m]
        testVal = mapData[i,j].adjusted_height
        testVal2 = mapData[neighbour in direction m].adjusted_height
        switch curBT:
            case ROAD, MAINHALL, WING:
                if testVal == (k + 1):        // ground level for this column
                    connectNodes(i, j, k, nbr, FOUNDATION, -1, testVal2 - testVal, 0, WALL_RAD_M + 1.0)
```

`connectNodes(i1,j1,k1, i2,j2,k2, conType, direction, heightDelta, axisLayer, wallRadInMeters, flags=0)` writes the connection at BOTH endpoint nodes (mirror), per VQ.

### Phase 5 — `E_BG_WING_TIPS`

`applyWingValues` per doc §3: for each axis-aligned wing run of length ≥ 2, find endpoints; set `BC_FLAG_WING_BEG` on one end, `BC_FLAG_WING_END` on the other; assign `wing_mult = 1.0` at endpoints tapering to smaller values inward (VQ's exact interpolation).

### Phase 6 — `E_BG_DOORS`

Per doc §3 "Phase detail: DOORS and windows":
```
for each WING connection at (i, j, k) in direction m:
    nbr = neighbour at m
    if ctClasses[mapData[nbr].connection_props[m]] == CTC_ROAD:
        upgrade to DOORWAY
    else:
        stay as WINDOWFRAME
        if direction == +1 (interior side):
            emit LANTERN connection on interior side
```

### Phase 7 — `E_BG_FILL_TERRAIN`

Top-down walk (per VQ doc §3 "The outer loop walks all nodes at all Z levels for each phase: k = (n == E_BG_FILL_TERRAIN) ? (terDataBufPitchZ - 1 - ktemp) : ktemp"):

For each tile column `(i, j)` descending from top:
- If any node at this (i, j, k) is WING/WINDOWFRAME/DOORWAY → mark tile-column as "building interior" starting at `adjusted_height + 1` and extending up until a wall caps it.
- Store in `block.building_interior_tex`: each byte is a bitmask of Y-levels inside rooms for that (x, z) pixel. 16 levels fit in a byte with 2 levels per bit pair, or we use R16UI if we need more precision.

### Phase 8 — Geometry emission

`building_emit_to_page(PageGeometry* out, Block* block, Page* page)`:
- Walk `building_data` tiles whose world (i, j) falls inside `page.world_min..world_max`.
- For each non-NONE connection on the foundation layer: emit `GeomEntry` superellipsoid with:
  - `bounds_min/max` = connection midpoint ± (WALL_RAD_M * wing_mult)
  - `power_vals.x` = 6.0 (near-cube)
  - `mat.material_id` = PLASTER for WING, STONE for FOUNDATION, WOOD for DOORWAY, GLASS for WINDOWFRAME, SHINGLE for top-level walls
- For each LANTERN connection: emit `LineEntry` with tangent along wall, `rad0 = 0.3`, `rad1 = 0.5`, `mat = METAL`.

## GPU Evaluation

`generate.comp.glsl` gains:

```glsl
layout(binding = 8) uniform usampler2D building_interior;

// In main(), after terrain + road override:
uvec4 interior_flags = texture(building_interior, hm_uv);
int   y_level = int(floor(world.y / 8.0));    // tile Y index
if (y_level >= 0 && y_level < 16 &&
    (interior_flags.r & (1u << uint(y_level))) != 0u) {
    result = vec4(0.0);   // voxel is inside a room — air
}
```

Bit i of the raster's byte at (x,z) = "voxel at Y-level i is inside a room at this (x,z) column".

## Per-Block / Per-Frame Lifecycle

1. `BlockPool.get_or_create`: allocate `building_data`, `map_data`, `building_interior_tex`. Set `buildings_dirty = true`.
2. `main.c3::drain_dirty`, inside `if (block.heightmap_dirty)` block:
   - `generate_heightmap_cpu`
   - `generate_roads_for_block` (now includes main-road overlay)
   - `buildings_generate_for_block(block, WORLD_SEED)` — runs phases 1–7, uploads `building_interior_tex`.
3. `world::geometry_generate_page`: append `building_emit_to_page(out, block, page)` after `plant_generate_page`.
4. `BlockPool.decrement_ref` (zero): free both buffers + destroy the texture.

## Testing

| Check | Expected |
|-------|----------|
| Build | `c3c build linux` green; all `$assert`s pass. |
| Visual | City blocks visible with rectangular lots bounded by main roads; each lot holds 1–3 small buildings with walls + doors facing roads + windows on other sides. |
| Height smoothing | No one-tile cliffs between adjacent building tiles. |
| Interior carve | Raymarch rays entering a building at top pass through roof → see air → see floor (roof visible above, floor visible below). |
| Determinism | Byte-identical building_data across two runs with the same WORLD_SEED. |
| Frame time | Within ±50% of M7.3 baseline. Building generation is O(grid_nodes) per block, runs once per block lifetime. |
| Memory | Block memory ~25 MB (heightmap_cpu + building_data + map_data + raster); × 4 blocks ≈ 100 MB CPU peak. |

## Risk Register

| Risk | Mitigation |
|------|------------|
| 65k building_data nodes × 310 B per block → 20 MB; 4 blocks = 80 MB | Acceptable for M7.4; revisit in M9+ if block pool grows. Consider packing `BuildingCon` tighter (drop `_pad`, use 16-bit fields). |
| 5-phase generation per block per first-dirty might be slow (hundreds of ms?) | One-shot per block lifetime, gated by `buildings_dirty`. If >300 ms, deferred-initialisation via a work-queue at end of `drain_dirty`. |
| `FILL_TERRAIN` in R8UI packs 8 Y-levels per byte but BUILDING_GRID_Y=16 | Use R16UI for the interior texture — 1 bit per Y-level per (x, z). Format plumbing confirmed in M7.3 (R8UI works; R16UI is a small format-enum addition). |
| `WING_TIPS` wing-run detection across the 3D grid O(grid³) worst case | Only runs along axis-aligned edges; practical complexity is O(grid² × wing_length). |
| Building-to-page intersection check | Skip optimisation for M7.4: walk all tiles per page. Page covers 16×16 tiles; per-page cost is 256 tile-checks × 12 connections = 3072 entry checks. Fine. |
| Shader gains 8th binding → UBO / SSBO count budget | 8 is well within typical driver limits (16+ texture units, 32+ SSBO). No issue. |
