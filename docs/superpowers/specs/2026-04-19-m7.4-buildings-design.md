**Status:** Spec

## Goal

Port VoxelQuest's building generator: a 3D `BuildingNode` grid whose 12 per-node connections are filled in across 5 phases (roads_and_buildings, wing_tips, doors, fill_terrain â€” basements skipped as in VQ). Walls, doors, windows, and lanterns emerge from the graph as `GeomEntry` / `LineEntry` records into the per-page SSBOs M7.1 set up. Completes the M7 "Done when" bar: a village of 5â€“10 buildings with recognisable walls, roofs, doorways, windows.

## Scope

### In
- 3D `BuildingNode` grid per block. **8 world units per tile**. Grid is `64 Ă— 64 Ă— 16` per block (~65k nodes Ă— ~310 B = ~20 MB per block; Ă—4 active blocks â‰ˆ 80 MB CPU).
- `BuildingCon` struct matching VQ with 12 connections per node (6 axial dirs Ă— 2 layers â€” foundation + superstructure). `ConType` enum (NONE/ROAD/MAINHALL/WING/FOUNDATION/DOORWAY/WINDOWFRAME/LANTERN).
- `MapNode[]` 2D layer (64Ă—64) for road-adjacency + adjusted-height bookkeeping.
- **Main-road overlay** added to M7.3's `generate_roads_for_block`: straight roads at fixed intervals (`BLOCK_SIZE_IN_LOTS` = 8 tiles) form city blocks around the maze. Matches VQ's `blockSizeInLots` behaviour.
- **Full 5-phase generation** (minus basements, per VQ):
  - `E_BG_ROADS_AND_BUILDINGS`: read mapData, extrude foundations under WING tiles, wall framework above.
  - `E_BG_WING_TIPS`: propagate `BC_FLAG_WING_BEG`/`_WING_END` along axis-aligned wing runs; set `wing_mult` taper value.
  - `E_BG_DOORS`: for each WING connection, check adjacent tile for ROAD â€” if yes upgrade to DOORWAY, else WINDOWFRAME; emit LANTERN on interior-side WINDOWFRAME.
  - `E_BG_FILL_TERRAIN`: top-down walk marks inside-room voxels as air.
- **Height consistency iteration** (up to 16 passes) over `mapData.adjusted_height`: clamp each tile to `max(neighbours) - 1`, enforce water-level floor. Prevents one-tile cliffs between adjacent building tiles.
- Geometry emission per page: walls â†’ superellipsoid `GeomEntry`, lanterns â†’ `LineEntry`, interior-carve texture.
- Flat roofs: top-level wall tile gets `Material.SHINGLE`. Pitched roofs deferred to follow-up task #20.

### Out (follow-ups)
- Pitched / gabled roofs (task #20).
- Basements (commented out in VQ).
- Multi-story interiors / stairs.
- Entity-picked buildings (M10 editor).
- Building palette variation beyond material swaps.

## Algorithms

### Phase 0 â€” Main-road overlay (extends M7.3)

In `src/world/road.c3::generate_roads_for_block` **before** `generate_maze`, if this block is a city:

```c3
int step = BLOCK_SIZE_IN_LOTS * BUILDING_TILE_PX;    // 64 world pixels
for (int z = 0; z < BLOCK_HEIGHTMAP_PIXELS; z += step) {
    draw_segment(&block.road, 0, z, BLOCK_HEIGHTMAP_PIXELS - 1, z, (char)ROAD_BIT);
}
for (int x = 0; x < BLOCK_HEIGHTMAP_PIXELS; x += step) {
    draw_segment(&block.road, x, 0, x, BLOCK_HEIGHTMAP_PIXELS - 1, (char)ROAD_BIT);
}
```

Creates a main-road grid inside the block. Maze fills between.

### Phase 1 â€” Build `map_data`

For each `(i, j)` in `BUILDING_GRID_X Ă— BUILDING_GRID_Z`:
1. Sample `block.road.pixels` at tile-centre pixel `(i * TILE + TILE/2, j * TILE + TILE/2)`. If `ROAD_BIT` â†’ `map.connection_props[*]` = `ROAD`.
2. Else, sample `block.heightmap_cpu` at tile centre â†’ `map.adjusted_height = floor(height_meters / TILE_PX)`.

### Phase 2 â€” Building selection

For each lot (bounded by main-road tiles), iterate eligible (non-road) interior tiles. Hash-driven per lot:
- Pick a rectangular WING footprint 2Ă—2 .. 3Ă—3 tiles somewhere inside the lot.
- Mark those tiles' `connection_props` = `WING`.
- One tile in the centre of each footprint becomes `MAINHALL` (drives door-hall orientation).

Lot bounds derived from main-road grid: each lot is `BLOCK_SIZE_IN_LOTS Ă— BLOCK_SIZE_IN_LOTS` tiles minus the surrounding road cells.

### Phase 2.5 â€” Building selection algorithm

Concrete: for each lot (at main-road intersections), run the following:

```
lot_seed = hash(block.cx, block.cz, world_seed, lot_i * 11 + lot_j * 13)
num_buildings = 1 + (lot_seed & 0x3)   // 1..4 per lot
for b in 0..num_buildings:
    b_seed = hash(lot_seed, b)
    foot_w = 2 + ((b_seed >>  0) & 0x1)   // 2 or 3
    foot_h = 2 + ((b_seed >>  8) & 0x1)   // 2 or 3
    pos_x  = lot_origin_x + 1 + ((b_seed >> 16) % (lot_w - foot_w - 1))
    pos_z  = lot_origin_z + 1 + ((b_seed >> 24) % (lot_h - foot_h - 1))

    // Skip if any tile in footprint is already ROAD or WING.
    if any tile in (pos_x..pos_x+foot_w, pos_z..pos_z+foot_h) has con != NONE:
        continue

    // Mark footprint: MAINHALL at centre, WING on the rest.
    cx = pos_x + foot_w/2
    cz = pos_z + foot_h/2
    for tile in footprint:
        if (tile.x, tile.z) == (cx, cz): connection_props[*] = MAINHALL
        else:                            connection_props[*] = WING
```

If the lot has fewer than 4Ă—4 non-road tiles (shouldn't happen with BLOCK_SIZE_IN_LOTS=8 minus 2 road lanes), skip the lot silently.

### Phase 3 â€” Height smoothing (up to 16 passes)

```
do {
    changed = false
    for each (i, j):
        m = map[i,j].adjusted_height
        p = max over 4 neighbours of map[nbr].adjusted_height
        if p - m > 1:
            map[i,j].adjusted_height = p - 1
            changed = true
        if newSeaLev > map[i,j].adjusted_height:
            map[i,j].adjusted_height = newSeaLev
            changed = true
    counter++
} while (changed && counter < BUILDING_HEIGHT_ITERS)
```

Clamps cliff deltas to 1 tile. `new_sea_lev = BUILDING_NEW_SEA_LEV = 2` tile-units (= y=16 world), defined as `const int BUILDING_NEW_SEA_LEV = 2;` in `building.c3`. Matches the road module's `WATER_LEVEL_Y = 16.0` in world units (16/TILE=8 â†’ 2 tile-units).

### Phase 4 â€” `E_BG_ROADS_AND_BUILDINGS`

Full triple loop (`i, j, k, m`) per VQ doc Â§3 "Phase detail":

```
for each tile (i, j, k):
    for each horizontal direction m in 0..3:
        curBT = mapData[i,j].connection_props[m]
        testVal = mapData[i,j].adjusted_height
        testVal2 = mapData[neighbour in direction m].adjusted_height
        switch curBT:
            case ROAD, MAINHALL, WING:
                if testVal == (k + 1):        // ground level for this column
                    connectNodes(i, j, k, nbr, FOUNDATION, -1, testVal2 - testVal, 0, WALL_RAD_M + 1.0)
```

`connect_nodes(i1,j1,k1, i2,j2,k2, con_type, direction, height_delta, axis_layer, wall_rad_m, flags=0)` writes the connection at BOTH endpoint nodes so that reads from either side see the same wall. Index math per VQ `f00352_gameblock.hpp`:

- For the endpoint at `(i1,j1,k1)`, the connection slot is `axis_layer * 6 + base_dir` where `base_dir = 0..5` is the axial direction `(i2-i1, j2-j1, k2-k1)` encoded as (+X=0, -X=1, +Y=2, -Y=3, +Z=4, -Z=5).
- For the endpoint at `(i2,j2,k2)`, the slot is `axis_layer * 6 + (base_dir ^ 1)` â€” the opposite direction.

Both endpoints receive identical `con_type`, `wall_rad_m`, `height_delta`; `direction` mirrors (each endpoint stores its own outward normal, so `direction` at endpoint 2 is negated).

### Phase 5 â€” `E_BG_WING_TIPS`

`apply_wing_values` per doc Â§3: for each axis-aligned wing run of length â‰¥ 2, find endpoints; set `wing_beg = true` on one end, `wing_end = true` on the other. `wing_mult` tapers **linearly** from `0.6` at each endpoint to `1.0` at the midpoint of the run (run length `n` â†’ position `i`'s mult = `0.6 + 0.4 * (1 - |i - (n-1)/2| / ((n-1)/2))`). The endpoint taper makes the tips read as "end of a wing" visually; the mid-run bulk stays full-thickness.

### Phase 6 â€” `E_BG_DOORS`

Per doc Â§3 "Phase detail: DOORS and windows":
```
for each WING connection at (i, j, k) in direction m:
    nbr = neighbour at m
    if ctClasses[mapData[nbr].connection_props[m]] == CTC_ROAD:
        upgrade to DOORWAY
    else:
        stay as WINDOWFRAME
        if direction == +1 (interior side):
            emit LANTERN connection on interior side
```

### Phase 7 â€” `E_BG_FILL_TERRAIN`

Top-down walk (per VQ doc Â§3 "The outer loop walks all nodes at all Z levels for each phase: k = (n == E_BG_FILL_TERRAIN) ? (terDataBufPitchZ - 1 - ktemp) : ktemp"):

For each tile column `(i, j)` descending from top:
- If any node at this (i, j, k) is WING/WINDOWFRAME/DOORWAY â†’ mark tile-column as "building interior" starting at `adjusted_height + 1` and extending up until a wall caps it.
- Store in `block.building_interior_tex` (R16UI, 512Ă—512): each 16-bit pixel is a bitmask where bit `k` == 1 means voxel at Y-level `k` is inside a room. Matches `BUILDING_GRID_Y = 16` exactly.

### Phase 8 â€” Geometry emission

`building_emit_to_page(PageGeometry* out, Block* block, Page* page)`:
- Walk `building_data` tiles whose world (i, j) falls inside `page.world_min..world_max`.
- For each non-NONE connection on the foundation layer: emit `GeomEntry` superellipsoid with:
  - `bounds_min/max` = connection midpoint Â± (WALL_RAD_M * wing_mult)
  - `power_vals.x` = 6.0 (near-cube)
  - `mat.material_id` = PLASTER for WING, STONE for FOUNDATION, WOOD for DOORWAY, GLASS for WINDOWFRAME, SHINGLE for top-level walls
- For each LANTERN connection: emit `LineEntry` with tangent along wall, `rad0 = 0.3`, `rad1 = 0.5`, `mat = METAL`.

## GPU Evaluation

`generate.comp.glsl` gains:

```glsl
layout(binding = 8) uniform usampler2D building_interior;

// In main(), after terrain + road override:
uvec4 interior_flags = texture(building_interior, hm_uv);
int   y_level = int(floor(world.y / 8.0));    // tile Y index
if (y_level >= 0 && y_level < 16 &&
    (interior_flags.r & (1u << uint(y_level))) != 0u) {
    result = vec4(0.0);   // voxel is inside a room â€” air
}
```

Bit i of the raster's byte at (x,z) = "voxel at Y-level i is inside a room at this (x,z) column".

## Per-Block / Per-Frame Lifecycle

1. `BlockPool.get_or_create`: allocate `building_data`, `map_data`, `building_interior_tex`. Set `buildings_dirty = true`.
2. `main.c3::drain_dirty`, inside `if (block.heightmap_dirty)` block:
   - `generate_heightmap_cpu`
   - `generate_roads_for_block` (now includes main-road overlay)
   - `buildings_generate_for_block(block, WORLD_SEED)` â€” runs phases 1â€“7, uploads `building_interior_tex`.
3. `world::geometry_generate_page`: append `building_emit_to_page(out, block, page)` after `plant_generate_page`.
4. `BlockPool.decrement_ref` (zero): free both buffers + destroy the texture.

## Risk Register

| Risk | Mitigation |
|------|------------|
| 65k building_data nodes Ă— 310 B per block â†’ 20 MB; 4 blocks = 80 MB | Acceptable for M7.4; revisit in M9+ if block pool grows. Consider packing `BuildingCon` tighter (drop `_pad`, use 16-bit fields). |
