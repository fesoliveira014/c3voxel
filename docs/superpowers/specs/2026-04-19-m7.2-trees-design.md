# M7.2 — Trees (L-system + Béziers)

**Status:** Spec
**Date:** 2026-04-19
**Predecessor:** M7.1 (paramArr + voronoi + materials, closed)
**Successors:** M7.3 (Roads), M7.4 (Buildings)
**Reference:** `docs/procedural-generation-vq.md` §4 (Trees)

## Goal

Port VoxelQuest's tree generator: a recursive L-system producing `GamePlantNode` trees, emitted as `TreeEntry`s (quadratic Béziers with optional leaf spheres) into the per-page geometry SSBO that M7.1 set up. First real consumer of the `TreeEntry[]` path; validates the Bézier evaluator branch of `generate.comp.glsl`.

## Scope

### In

- Six `PlantRules` variants ported verbatim from VoxelQuest (`OAK/OAK2/BARE_OAK × TRUNK/ROOTS`).
- `PlantRules` table in C3 with the exact numeric values from VQ `f00341_gameplant.hpp`.
- `apply_rules` recursion producing a `GamePlantNode` tree up to `MAX_PLANT_GEN = 6` depth.
- Tree-to-entry flattener: each node emits one `TreeEntry` with quadratic Bézier control points + optional leaf sphere.
- Deterministic per-page placement: 3–8 trees per page scattered by hash(page.px, page.pz, world_seed); filtered by a simple heightmap floor (`y > 16`, any higher is a post-M7.2 concern).
- GPU evaluator: VQ's tangent-line Bézier approximation + leaf-sphere inside-test. Shader's `eval_tree_bezier` stub gets replaced.

### Out

- Poisson-disk scatter (polish; M7.4 or later).
- Terrain-aware placement beyond the `y > 16` floor (M7.4 reads roads, which influences tree placement).
- Seasonal variations, wind animation, L-system variants beyond the 6 VQ ones.
- Per-branch LOD (all trees evaluate every branch regardless of viewer distance).
- Roots being visible — they're generated since the spec calls for all 6 variants but iso raymarch hits terrain first, so roots are buried under terrain surface. They exist in the SSBO but won't render. See §Known Limits.

`TreeEntry` from M7.1 is reused verbatim. Field mapping for trees (VQ doc §4, `initTree(&tempVec, &tempVec2, &tempVec3)`):
- `p0` = **parent midpoint** (the curve's starting anchor).
- `p1` = **current node midpoint** (Bézier control point that bends the curve).
- `p2` = **current node begPoint** (== parent's end_point; the junction vertex).
- `thick_vals.x` = begThickness, `thick_vals.y` = endThickness (world units), `thick_vals.z` = sphereRad (0 if no leaf at this branch), `thick_vals.w` = **leaf material id** (0 if no leaf).
- `mat.material_id` = branch material; other `MaterialParams` fields unchanged (kept entry-type-agnostic).
- `vis_min/vis_max` = AABB of (p0, p1, p2) expanded by `max(begThickness, endThickness, sphereRad)`.

### `src/world/plant_gen.c3` (new)

Live on the CPU only; never on the GPU.

```c3
module c3voxel::world;

import std::collections::list;

import c3voxel::core;
import c3voxel::voxel;

const int MAX_NODES_PER_TREE = 1024;

// Scratch node used during L-system recursion. Children are indices into the
// owning List{PlantNode}, not pointers — lets List.push reallocate without
// invalidating parent→child links.
struct PlantNode {
    core::Vec3 beg_point;
    core::Vec3 end_point;
    core::Vec3 tangent;            // unit vector
    core::Vec3 base_shoot;          // orthogonal to tangent; rotated to place children
    float      beg_thickness;
    float      end_thickness;
    float      sphere_rad;           // 0 if no leaf at this node
    int        first_child;          // index into owning list, -1 = leaf
    int        num_children;          // 0 at leaves
    bool       is_root_node;          // true for the virtual "seed" parent
}

alias PlantNodeList = List {PlantNode};

// Grow one tree at `origin` of the given type; flatten every non-root node
// into a TreeEntry and push into `out.tree`. `scratch` is a caller-owned
// List{PlantNode}, cleared at the start of the call. Caps growth at
// MAX_NODES_PER_TREE with a one-shot eprintfn warning. `tree_seed` is the
// only source of randomness in `apply_rules` (counter-stepped per call),
// so two runs with the same seed produce byte-identical output.
fn void plant_grow_tree(
    PageGeometry*      out,
    PlantNodeList*     scratch,
    voxel::PlantType   type,
    core::Vec3         origin,
    uint               tree_seed);

// Place 3–8 trees on the page based on hash(page.px, page.pz, world_seed),
// sample the block heightmap at each (wx, wz), then plant_grow_tree each.
// Reuses one scratch list across trees.
fn void plant_generate_page(
    PageGeometry*   out,
    Page*           page,
    Block*          block,
    uint            world_seed,
    PlantNodeList*  scratch);
```

### Determinism

Every random value consumed by `apply_rules` comes from a counter-stepped
sub-hash of `tree_seed`. No global RNG. Helper:

```c3
fn float plant_frand(uint* counter, uint base_seed) {
    uint h = voxel::voronoi_hash(base_seed, 0u, 0u, *counter);  // reuse xorshift hash from M7.1
    *counter += 1;
    return voxel::voronoi_frand(h);   // returns [0, 1)
}
fn float plant_frand_range(uint* counter, uint base_seed, float lo, float hi) {
    return lo + (hi - lo) * plant_frand(counter, base_seed);
}
```

Each `plant_grow_tree` call initialises a local `uint counter = 0` and passes `&counter + tree_seed` through the recursion. Two runs with the same seed consume identical hash sub-streams → identical output.

### `max_length` computation

Total path length used for linear thickness lerp:
```c3
max_length = sum(rules.branch_length[0 .. rules.num_generations - 1])
```
Worked example (OAK_TRUNK, num_generations=2, branch_length=[6, 8]): `max_length = 6 + 8 = 14`. At generation 1, a child spawned after the parent's length-6 branch has `tot_length=6`, `lerp_start = 6/14 ≈ 0.43`, `lerp_end = 14/14 = 1.0`. Its thickness tapers from mix(1.0, 0.4, 0.43) = 0.742 to mix(1.0, 0.4, 1.0) = 0.4 — matching VQ visual.

**Recursion (mirrors VQ `applyRules`):**

```
apply_rules(rules, parent_idx, cur_gen, tot_length, max_length, seed, counter_ref):
    parent = scratch[parent_idx]
    num_kids = int(plant_frand_range(counter, seed, rules.num_children[0], rules.num_children[1]))
    parent.first_child = scratch.len()
    parent.num_children = num_kids
    for i in 0..num_kids:
        jitter = plant_frand(counter, seed) - 0.5   // [-0.5, 0.5)
        theta  = (i / num_kids) * 2π + jitter * 2π * (1 - rules.angle_uniformity_u) / num_kids
        child.beg_point = parent.end_point
        child.end_point = parent.end_point + rotate(parent.base_shoot, parent.tangent, theta) * curLength
        child.tangent   = update_tangent(parent, plant_frand_range(counter, seed, rules.divergence_angle_v[0], rules.divergence_angle_v[1]))

        lerp_start = tot_length / max_length
        lerp_end   = (tot_length + curLength) / max_length
        child.beg_thickness = mix(rules.beg_thickness, rules.end_thickness, lerp_start)
        child.end_thickness = mix(rules.beg_thickness, rules.end_thickness, lerp_end)
        child.sphere_rad    = (cur_gen == rules.sphere_gen) ? rules.sphere_rad : 0
        if scratch.len() >= MAX_NODES_PER_TREE: warn_and_return
        scratch.push(child)
    for i in 0..num_kids:
        if cur_gen < rules.num_generations - 1:
            apply_rules(rules, parent.first_child + i, cur_gen + 1,
                        tot_length + curLength, max_length, seed, counter_ref)
```

**Flatten:** post-recursion, walk the node list index-by-index and push one `TreeEntry` per node whose `is_root_node` is false. Each entry's Bézier uses:
- `p0 = midpoint(parent.beg_point, parent.end_point)`  (via `eval_bezier(0.5)` at CPU flatten time — avoids chord-midpoint skew for high-curvature nodes)
- `p1 = midpoint(node.beg_point, node.end_point)`      (the bend control point)
- `p2 = node.beg_point`                                 (== parent.end_point — the junction)

### Placement

```c3
fn void plant_placement_for_page(
    Page*          page,
    uint           world_seed,
    core::Vec3[8]  out_origins,        // up to 8 trees
    voxel::PlantType[8] out_types,
    int*           out_count)
{
    uint h = hash4(page.px, page.pz, world_seed, 0);
    int count = 3 + (int)((h >> 24) & 0x5);   // 3..8 directly, no clamp

    for i in 0..count:
        uint h2 = hash4(page.px, page.pz, world_seed, i);
        float fx = (h2 & 0xFFFF) / 65535.0;
        float fz = ((h2 >> 16) & 0xFFFF) / 65535.0;
        float wx = mix(page.world_min.x, page.world_max.x, fx);
        float wz = mix(page.world_min.z, page.world_max.z, fz);
        // Heightmap-driven elevation: sample block.heightmap (CPU-side from M6)
        // at nearest-neighbour of this tree's (wx, wz). Skip tree if below water.
        float wy = block_heightmap_sample(block, wx, wz);
        if (wy < WATER_LEVEL_Y) continue;

        uint type_h = hash4(page.px, page.pz, world_seed, i + 100);
        voxel::PlantType type = (voxel::PlantType)(type_h % (uint)voxel::PLANT_TYPE_COUNT);
        // prefer trunk variants 3:1 over roots so surface density matches visual expectation
        if (type == voxel::PlantType.OAK_ROOTS   ||
            type == voxel::PlantType.OAK2_ROOTS  ||
            type == voxel::PlantType.BARE_OAK_ROOTS) {
            type = (voxel::PlantType)((type_h >> 8) % 3u * 2u);  // snap to nearest trunk variant
        }

        out_origins[i] = { wx, wy, wz };
        out_types[i]   = type;
    }
    *out_count = count;
}
```

## GPU Evaluation

### `eval_tree_bezier` in `resources/shaders/generate.comp.glsl`

Replace M7.1's magenta stub with:

```glsl
vec4 eval_tree_bezier(vec3 w, TreeEntry e) {
    vec3 p0 = e.p0.xyz;
    vec3 p1 = e.p1.xyz;
    vec3 p2 = e.p2.xyz;
    float beg_t       = e.thick_vals.x;
    float end_t       = e.thick_vals.y;
    float sph         = e.thick_vals.z;
    int   leaf_mat_id = int(e.thick_vals.w);

    // VQ tangent-line approximation (doc §4). Estimate t by projecting onto
    // the P0->P2 chord, then evaluate the exact Bezier at that t.
    vec3 chord = p2 - p0;
    float chord_sq = max(dot(chord, chord), 1e-6);
    float t = clamp(dot(w - p0, chord) / chord_sq, 0.0, 1.0);
    float one_minus_t = 1.0 - t;
    vec3 on_bezier = one_minus_t * one_minus_t * p0
                   + 2.0 * one_minus_t * t * p1
                   + t * t * p2;
    float d_branch = distance(w, on_bezier);
    float thickness = mix(beg_t, end_t, t);

    if (d_branch < thickness) {
        return material_color(int(e.mat.x));   // branch material (mat.x = material_id)
    }
    if (sph > 0.0 && distance(w, p2) < sph) {
        return material_color(leaf_mat_id);    // leaf material from thick_vals.w
    }
    return vec4(0.0);
}
```

**Vis-AABB:** `aabb_contains(world, e.vis_min.xyz, e.vis_max.xyz)` from the M7.1 nearest-match pass already culls entries whose bounding box doesn't contain the voxel. Tree entries get bounding boxes generously expanded (`max(begThickness, endThickness, sphereRad)`).

**Nearest-match behaviour:** the tree branch is a 3D shape. `center_point` for nearest-match comparison = midpoint of `p0, p2`, which VQ uses for corner detection. Good enough for M7.2; refine in M7.4 if foliage-intersection artifacts appear.

## Implementation Order

1. **`PlantRules` table + `PlantType` enum.** Port VQ numbers verbatim; zero callsites. Commit.
2. **`PlantNode` scratch struct + `apply_rules` recursion.** Pure CPU, testable in isolation. Commit.
3. **Tree-to-entry flattener.** Walk the node tree, push `TreeEntry`s. Commit.
4. **Per-page placement hash.** Emit tree origins + types. Commit.
5. **Hook `plant_generate_page` into `geometry_generate_page`.** Trees now populate the tree SSBO every drain. Commit.
6. **GPU `eval_tree_bezier` implementation.** Replace the magenta stub. Commit.

Each step ends with `c3c build linux` and, for steps 5 + 6, a runtime smoke that trees render above the heightmap.

## Known Limits

- **Roots invisible.** All 6 variants are generated, but iso raymarch hits terrain before reaching y<0, so roots (`OAK_ROOTS` etc.) don't render. They still occupy the SSBO — keep the entries for future terrain-transparency features. Filtering them out during emission is a M7.4 decision.
- **Water level constant `WATER_LEVEL_Y = 16.0`.** Trees skip placement below this. Heightmap-driven placement is resolved via `block_heightmap_sample`; biome-aware rules (forest vs desert) are M7.4 scope.
