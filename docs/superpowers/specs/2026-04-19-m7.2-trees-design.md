# M7.2 — Trees (L-system + Béziers)

**Status:** Spec
**Date:** 2026-04-19
**Predecessor:** M7.1 (paramArr + voronoi + materials, closed)
**Successors:** M7.3 (Roads), M7.4 (Buildings)
**Reference:** `docs/procedural-generation-vq.md` §4 (Trees)

## Goal

Port VoxelQuest's tree generator: a recursive L-system producing `GamePlantNode` trees, emitted as `TreeEntry`s (quadratic Béziers with optional leaf spheres) into the per-page geometry SSBO that M7.1 set up. First real consumer of the `TreeEntry[]` path; validates the Bézier evaluator branch of `generate.comp.glsl`.

## Scope

### In

- Six `PlantRules` variants ported verbatim from VoxelQuest (`OAK/OAK2/BARE_OAK × TRUNK/ROOTS`).
- `PlantRules` table in C3 with the exact numeric values from VQ `f00341_gameplant.hpp`.
- `apply_rules` recursion producing a `GamePlantNode` tree up to `MAX_PLANT_GEN = 6` depth.
- Tree-to-entry flattener: each node emits one `TreeEntry` with quadratic Bézier control points + optional leaf sphere.
- Deterministic per-page placement: 3–8 trees per page scattered by hash(page.px, page.pz, world_seed); filtered by a simple heightmap floor (`y > 16`, any higher is a post-M7.2 concern).
- GPU evaluator: VQ's tangent-line Bézier approximation + leaf-sphere inside-test. Shader's `eval_tree_bezier` stub gets replaced.

### Out

- Poisson-disk scatter (polish; M7.4 or later).
- Terrain-aware placement beyond the `y > 16` floor (M7.4 reads roads, which influences tree placement).
- Seasonal variations, wind animation, L-system variants beyond the 6 VQ ones.
- Per-branch LOD (all trees evaluate every branch regardless of viewer distance).
- Roots being visible — they're generated since the spec calls for all 6 variants but iso raymarch hits terrain first, so roots are buried under terrain surface. They exist in the SSBO but won't render. See §Known Limits.

`TreeEntry` from M7.1 is reused verbatim. Field mapping for trees:
- `p0, p1, p2` — Bézier control points. Per VQ convention: `p0` = parent midpoint, `p1` = current midpoint, `p2` = current begPoint.
- `thick_vals.x` = begThickness, `thick_vals.y` = endThickness (world units), `thick_vals.z` = sphereRad (0 if no leaf at this branch).
- `mat.material_id` = branch material; `mat.normal_id` = leaf material (repurposed for this; see §Notes).
- `vis_min/vis_max` = AABB of (p0, p1, p2) expanded by `max(begThickness, endThickness, sphereRad)`.

### `src/world/plant_gen.c3` (new)

Live on the CPU only; never on the GPU.

```c3
module c3voxel::world;

import c3voxel::core;
import c3voxel::voxel;

// Scratch node used during L-system recursion. Flattened into TreeEntry at the end.
struct PlantNode {
    core::Vec3 beg_point;
    core::Vec3 end_point;
    core::Vec3 tangent;            // unit vector
    core::Vec3 base_shoot;          // orthogonal to tangent, used to rotate children
    float      beg_thickness;
    float      end_thickness;
    float      sphere_rad;           // 0 if no leaf at this node
    int        num_children;          // 0 at leaves
    PlantNode* children;              // length == num_children
    bool       is_root_node;          // true for the virtual "seed" parent
}

// Grow one tree at `origin` of the given type; write all emitted branches into `out`.
// Uses a scratch arena for PlantNode allocation so we don't fragment the heap.
fn void plant_grow_tree(
    PageGeometry*      out,
    voxel::PlantType   type,
    core::Vec3         origin,
    uint               tree_seed);

// Place 3–8 trees on the page based on hash(page.px, page.pz, world_seed),
// filter below-water by heightmap sample, call plant_grow_tree for each.
fn void plant_generate_page(
    PageGeometry* out,
    Page*          page,
    uint           world_seed);
```

**Recursion (mirrors VQ `applyRules`):**

```
apply_rules(rules, parent, cur_gen, max_gen, tot_length, max_length):
    for i in 0..parent.num_children:
        child = &parent.children[i]
        child.beg_point = parent.end_point
        theta = (i / num_children) * 2π + jitter * 2π * (1 - angleUniformityU) / num_children
        child.end_point = parent.end_point + rotate_around_tangent(base_shoot, parent.tangent, theta) * curLength
        child.tangent = update_tangent(parent, random_in(rules.divergence_angle_v))

        lerp_start = tot_length / max_length
        lerp_end   = (tot_length + curLength) / max_length
        child.beg_thickness = mix(beg_thickness, end_thickness, lerp_start)
        child.end_thickness = mix(beg_thickness, end_thickness, lerp_end)
        child.sphere_rad = (cur_gen == sphere_gen) ? rules.sphereRad : 0

        if cur_gen < max_gen:
            apply_rules(rules, child, cur_gen + 1, max_gen, tot_length + curLength, max_length)
```

**Flatten:** post-recursion, walk the node tree and push one `TreeEntry` per node whose `parent != root`. Each entry's Bézier uses `p0 = parent_midpoint`, `p1 = parent_end_point`, `p2 = node_midpoint`.

### Placement

```c3
fn void plant_placement_for_page(
    Page*          page,
    uint           world_seed,
    core::Vec3[8]  out_origins,        // up to 8 trees
    voxel::PlantType[8] out_types,
    int*           out_count)
{
    uint h = hash4(page.px, page.pz, world_seed, 0);
    int count = 3 + (int)((h >> 24) & 0x7);   // 3..10, clamp to [3, 8]
    if (count > 8) count = 8;

    for i in 0..count:
        uint h2 = hash4(page.px, page.pz, world_seed, i);
        float fx = (h2 & 0xFFFF) / 65535.0;
        float fz = ((h2 >> 16) & 0xFFFF) / 65535.0;
        float wx = mix(page.world_min.x, page.world_max.x, fx);
        float wz = mix(page.world_min.z, page.world_max.z, fz);
        float wy = 16.0;    // TEMP water-level; proper heightmap sampling post-M7.2

        uint type_h = hash4(page.px, page.pz, world_seed, i + 100);
        voxel::PlantType type = (voxel::PlantType)(type_h % 6);
        // prefer trunk variants 3:1 over roots so surface density matches visual expectation
        if (type == voxel::PlantType.OAK_ROOTS   ||
            type == voxel::PlantType.OAK2_ROOTS  ||
            type == voxel::PlantType.BARE_OAK_ROOTS) {
            type = (voxel::PlantType)((type_h >> 8) % 3 * 2);  // snap to nearest trunk variant
        }

        out_origins[i] = { wx, wy, wz };
        out_types[i]   = type;
    }
    *out_count = count;
}
```

## GPU Evaluation

### `eval_tree_bezier` in `resources/shaders/generate.comp.glsl`

Replace M7.1's magenta stub with:

```glsl
vec4 eval_tree_bezier(vec3 w, TreeEntry e) {
    vec3 p0 = e.p0.xyz;
    vec3 p1 = e.p1.xyz;
    vec3 p2 = e.p2.xyz;
    float beg_t = e.thick_vals.x;
    float end_t = e.thick_vals.y;
    float sph   = e.thick_vals.z;

    // VQ tangent-line approximation (doc §4). Estimate t by projecting onto
    // the P0->P2 chord, then evaluate the exact Bezier at that t.
    vec3 chord = p2 - p0;
    float chord_sq = max(dot(chord, chord), 1e-6);
    float t = clamp(dot(w - p0, chord) / chord_sq, 0.0, 1.0);
    float one_minus_t = 1.0 - t;
    vec3 on_bezier = one_minus_t * one_minus_t * p0
                   + 2.0 * one_minus_t * t * p1
                   + t * t * p2;
    float d_branch = distance(w, on_bezier);
    float thickness = mix(beg_t, end_t, t);

    if (d_branch < thickness) {
        return material_color(int(e.mat.x));   // branch material
    }
    if (sph > 0.0 && distance(w, p2) < sph) {
        return material_color(int(e.mat.y));   // leaf material (mat.y holds leaf id)
    }
    return vec4(0.0);
}
```

**Vis-AABB:** `aabb_contains(world, e.vis_min.xyz, e.vis_max.xyz)` from the M7.1 nearest-match pass already culls entries whose bounding box doesn't contain the voxel. Tree entries get bounding boxes generously expanded (`max(begThickness, endThickness, sphereRad)`).

**Nearest-match behaviour:** the tree branch is a 3D shape. `center_point` for nearest-match comparison = midpoint of `p0, p2`, which VQ uses for corner detection. Good enough for M7.2; refine in M7.4 if foliage-intersection artifacts appear.

## Per-Page Upload Flow

```c3
fn void geometry_generate_page(PageGeometry* out, Page* page, uint world_seed) {
    out.clear();

    // M7.1 smoke superellipsoids stay for now; M7.4 replaces them with buildings.
    geometry_emit_smoke_boxes(out, page);

    // M7.3 will add roads, M7.4 buildings + lanterns.
}
```

`main.c3::drain_dirty` already invokes this once per dirty page; no change needed beyond updating the function name.

## Implementation Order

1. **`PlantRules` table + `PlantType` enum.** Port VQ numbers verbatim; zero callsites. Commit.
2. **`PlantNode` scratch struct + `apply_rules` recursion.** Pure CPU, testable in isolation. Commit.
3. **Tree-to-entry flattener.** Walk the node tree, push `TreeEntry`s. Commit.
4. **Per-page placement hash.** Emit tree origins + types. Commit.
5. **Hook `plant_generate_page` into `geometry_generate_page`.** Trees now populate the tree SSBO every drain. Commit.
6. **GPU `eval_tree_bezier` implementation.** Replace the magenta stub. Commit.

Each step ends with `c3c build linux` and, for steps 5 + 6, a runtime smoke that trees render above the heightmap.

## Known Limits

- **Roots invisible.** All 6 variants are generated, but iso raymarch hits terrain before reaching y<0, so roots (`OAK_ROOTS` etc.) don't render. They still occupy the SSBO — keep the entries for future terrain-transparency features. Filtering them out during emission is a M7.4 decision.
- **Water level hardcoded at y=16.** Heightmap-driven placement is M7.4 scope; this milestone is the L-system + Bézier + placement plumbing, not the environmental logic.
