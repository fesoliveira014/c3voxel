## Phase 1 Design

### Todo 4 — `TextureSlot` registry (gpu/texture.c3)

**Problem:** `cmd.bind_image_2d(b, tex, access, format)` takes a `Format` arg the caller has to supply, creating a source of truth split between creation time and bind time. Desyncs silently produce wrong output.

**Fix:** opaque handle with per-handle registry, following the existing `ShaderSlot` pattern in `src/gpu/shader.c3`.

```c3
struct TextureSlot @local {
    uint      gl_id;         // 0 = slot free
    Format    fmt;
    Pixel     width;
    Pixel     height;
    Pixel     depth;          // 1 for 2D, actual depth for 3D
    int       levels;
    bool      is_3d;
    char      generation;
}

const usz MAX_TEXTURES = 256;
TextureSlot[MAX_TEXTURES] texture_slots @local;
```

`Texture2D` and `Texture3D` stay as distinct `typedef`s on top of `uint`, but the uint is now a packed handle (`core::handle_pack(idx, gen)`), not a raw GL id.

**API changes:**
- `texture2d_create` / `texture3d_create` — allocate slot, store format, return handle.
- `texture2d_destroy` / `texture3d_destroy` — bump generation, free GL object, clear slot. **Caller must unbind from any active image/sampler unit first** (same contract as current code; documented here explicitly).
- New internal: `texture_gl_id(handle) -> uint`, `texture_format(handle) -> Format`.
- `Commands.bind_texture(binding, Texture2D)` / `bind_texture_3d(binding, Texture3D)` — resolve handle to GL id internally.
- `Commands.bind_image_2d(binding, Texture2D, Access)` — `Format` arg removed; looked up from slot.
- `Commands.bind_image_3d(binding, Texture3D, Access)` — same.

**Framebuffer attachments go through the registry too.** `Framebuffer.colors[]` is already `Texture2D[MAX_FBO_ATTACHMENTS]` (`src/gpu/resource.c3:24`). `framebuffer_create` already calls `texture2d_create` for each color and the depth (`src/gpu/framebuffer.c3:29,44`), so attachments land in the registry automatically. `framebuffer_destroy` must call `texture2d_destroy` on each attachment (not `glDeleteTextures` directly) so generations bump. Spec explicitly enforces this.

**Callers touched (drop Format args):** `src/voxel/raymarch.c3`, `src/voxel/generate.c3`, `src/voxel/distance_field.c3`, `src/render/composite.c3`, `src/gpu/framebuffer.c3`, and `src/main.c3` if it binds directly.

**Invalid handles:** `TEXTURE2D_INVALID` / `TEXTURE3D_INVALID` stay as `(TypeN)0`. Slot 0 is reserved as the invalid slot (`gl_id = 0`, `generation = 0`); lookups return `gl_id = 0`, i.e. "unbind." `handle_index` of 0 therefore always resolves to the reserved slot regardless of generation.

### Todo 3 — `$feature(HOT_RELOAD)` gating

**Problem:** shader hot-reload stat-polls every file every frame, leaking into release builds.

**Fix:** compile-time `$feature(HOT_RELOAD)`. Default target (`c3voxel`) enables it; new `linux-release` target disables it.

**project.json** — add `features` to existing `linux` and `windows` targets, plus new release targets:
```json
"targets": {
  "windows":       { ..., "features": ["HOT_RELOAD"] },
  "linux":         { ..., "features": ["HOT_RELOAD"] },
  "linux-release": {
    "type": "executable",
    "name": "c3voxel",
    "target": "linux-x64",
    "features": [],
    "opt": "O3"
  },
  "windows-release": { ..., "features": [], "opt": "O3" }
}
```
Note: existing targets are `windows` and `linux`, not `c3voxel`. The executable *name* is `c3voxel`; the *target* names are platform-specific.

**src/gpu/shader.c3:**
```c3
fn bool shader_reload_if_changed(Shader s) {
    $if !$feature(HOT_RELOAD):
        return false;
    $endif
    // existing body unchanged
}
```

`src/main.c3` keeps the call site; no per-frame branch in C3 — the feature flag compiles the body out entirely.

### Todo 6 — Hierarchical GPU timer (gpu/timer.c3 new)

**Problem:** no visibility into per-stage GPU cost.

**Fix:** stack-based `time_push/pop` producing a tree; resolve one frame late to avoid stalls.

```c3
const int MAX_TIMER_NODES = 256;
const int MAX_TIMER_DEPTH = 8;
const int TIMER_FRAMES    = 2;   // double-buffered queries

alias NodeIndex = int;
const NodeIndex NODE_NONE = -1;
alias FrameIndex = int;

struct TimerNode {
    String    label;
    uint      query_start;      // GL_TIMESTAMP, allocated from pool at push
    uint      query_end;
    NodeIndex parent;            // NODE_NONE = root
    NodeIndex first_child;       // NODE_NONE = no children yet
    NodeIndex next_sibling;      // NODE_NONE = last sibling
    double    gpu_ms;            // filled by resolve_prev
}

struct Timer {
    TimerNode[MAX_TIMER_NODES]                 nodes;
    int                                         node_count;
    NodeIndex[MAX_TIMER_DEPTH]                  stack;
    int                                         stack_top;     // -1 = empty
    uint[TIMER_FRAMES * MAX_TIMER_NODES * 2]    query_pool;    // GL query names
    FrameIndex                                  frame_index;
    bool                                        overflow_warned;
}

fn void Timer.init(&self);
fn void Timer.destroy(&self);
fn void Timer.begin_frame(&self);          // reset tree
fn void Timer.end_frame(&self);            // insert end queries
fn void Timer.resolve_prev(&self);         // read prev frame queries
fn TimerNode* Timer.root(&self);           // for HUD traversal later

fn void Commands.time_push(&self, String label);   // push node, insert start query
fn void Commands.time_pop(&self);                   // insert end query, pop
```

**Overflow behaviour:**
- Exceeding `MAX_TIMER_NODES` in a frame: silently drops the push (returns a sentinel node), pops still decrement stack_top; log a one-shot warning via `overflow_warned`.
- Exceeding `MAX_TIMER_DEPTH`: same — drop the push, log once.
- `Timer.init` calls `gl::genQueries(TIMER_FRAMES * MAX_TIMER_NODES * 2, &query_pool[0])`. `Timer.destroy` calls `gl::deleteQueries`.

**Main loop integration:**
```c3
cmd.time_push("frame");
cmd.time_push("stream");   streaming_update(...); cmd.time_pop();
cmd.time_push("generate"); drain_dirty(...);      cmd.time_pop();
cmd.time_push("composite");composite.dispatch(...); cmd.time_pop();
cmd.time_pop();
timer.end_frame();
timer.resolve_prev();
```

Console dump on F12 (or similar key) formats the tree; HUD wiring is M8.

### Todo 10 — UBO std140 convention

**Problem:** four UBOs (`GenerateUniforms`, `RayMarchUniforms`, `CompositeUniforms`, JFA uniforms) hand-manage std140 padding. Silent breakage if a field is reordered or a `Vec3` sneaks in.

**Fix:** document convention + assert. Deferred to macro when a 5th UBO lands.

**New file `docs/ubo-convention.md`** — rules:
- `@packed @align(16)` on the struct.
- Only `core::Vec4`, `core::IVec4`, `float[N]` pad, scalar `int`/`float`.
- No `core::Vec3` — pad to `Vec4` or split.
- Arrays only of `Vec4` / `IVec4`.
- Trailing `float[2] _pad` / `float[3] _pad` if total not 16-aligned.

**At each UBO struct's module:**
```c3
$assert(MyUniforms.sizeof % 16 == 0);
```

And a comment above each UBO:
```c3
// std140 layout — see docs/ubo-convention.md.
// Promotion path: if a 5th UBO lands, convert to an @std140 macro that
// walks fields and emits alignment asserts automatically.
struct CompositeUniforms @packed @align(16) { ... }
```

## Phase 2 Design — True-Iso Raymarch (todo 12)

### Problem

Current raymarch (M6 deviation D1) projects each page top-down and writes a square sub-rect of the holder FBO. The iso camera then displays squares, not the parallelogram-shaped iso sprites the architecture calls for. Pan across holder boundaries shows axis-aligned edges instead of iso-aligned.

### Fix

**One dispatch per holder**, 4 page `sampler3D` bindings, per-pixel AABB selection. The holder FBO is rendered in iso projection directly.

**Shader (`resources/shaders/raymarch.comp.glsl`) — rewrite:**

```glsl
layout(binding = 0) uniform sampler3D   page_volumes[4];     // voxel data per page
layout(binding = 4) uniform usampler3D  dist_field;           // R8UI 3D DF (matches current)
layout(rgba8, binding = 5) uniform writeonly image2D  color_out;
layout(r16f,  binding = 6) uniform writeonly image2D  height_out;

layout(std140, binding = 7) uniform U {
    vec4  camera_forward;            // iso forward, world-space (normalized)
    vec4  holder_world_min;           // holder aabb for ray-start derivation
    vec4  holder_world_max;
    vec4  page_world_min[4];
    vec4  page_world_max[4];
    ivec4 resolution;                 // fbo width, fbo height, 0, 0
    ivec4 pitches;                    // volume pitch, df pitch, cell_voxels, 0
    ivec4 write_offset;               // sub-rect origin into holder fbo (x,y,0,0)
    ivec4 page_valid_mask;            // bit i set = page_volumes[i] is resident
    float iso_zoom;
    float iso_tilt;
    vec2  _pad0;
    // struct aligns to 16; _pad0 closes to 256 bytes.
};
```

**Size math:** 3×vec4 + 2×vec4[4] + 4×ivec4 + 2 float + vec2 = 48 + 128 + 64 + 8 + 8 = 256 bytes. `256 % 16 == 0` ✓.

Per pixel:
1. Invert iso: pixel (x,y) + holder origin → world ray origin on the "top" iso plane above the holder AABB.
2. Ray direction: `camera_forward`.
3. For t in increasing order of AABB entry:
   - Find nearest unvisited page AABB that ray intersects.
   - March that page's sampler3D from entry to exit, sampling DF for large skips.
   - On voxel hit: write color + height, return.
4. Miss: write transparent + far height.

**C3 (`src/voxel/raymarch.c3`) changes:**

- Delete `View` struct, `dispatch_into`, and the free-cam `dispatch` entry. The top-down path was only ever a M6 stop-gap (D1 in `docs/decisions/m6-deferrals.md`) — iso is now the only path.
- `dispatch_raw` is deleted with `dispatch_into` (free-cam debug is not needed; restore from git if ever required).
- New public entry: `dispatch_holder`. `RayMarchContext.target` and `height_target` members also deleted — output images are the per-holder FBO attachments now.
- New:
  ```c3
  fn void RayMarchContext.dispatch_holder(
      &self, gpu::Commands* cmd,
      gpu::Texture3D[4] page_volumes,
      DistanceField* df,
      Holder* h,
      Page*[4] pages,
      gpu::Framebuffer* holder_fbo,
      core::IVec4 page_valid_mask,
  );
  ```

- New uniforms struct mirroring the shader layout above (including `IVec4 page_valid_mask`).

**Partial residency:** during streaming, a holder may have fewer than 4 resident pages (eviction or initial fault-in). `dispatch_holder`'s caller must:
1. Bind `gpu::TEXTURE3D_INVALID` for missing pages (GL sampler unit gets texture 0).
2. Set the bit in `page_valid_mask` only for pages whose ring slot is populated this frame.

Shader checks `page_valid_mask` before each AABB test — invalid pages are skipped. A holder with zero valid pages writes transparent pixels (no-op useful early-out).

**Alternative callable precondition:** `drain_dirty` only dispatches holders with all 4 pages resident. Simpler, but stalls visual updates until streaming is complete. We go with the per-pixel mask instead so partial holders still render correctly during streaming.

**Main-loop change (`drain_dirty` in `src/main.c3`):**

Per `docs/decisions/m6-deferrals.md` §D3, the dirty queue is already **holder-granular** (one `DirtyEntry` per holder, re-queued if any page dirties). So the main loop need not collapse — iterate dirty holders directly.

```c3
// drained: holder ids whose pages need regen this frame
for (int i = 0; i < drained.len; i++) {
    HolderId hid = drained[i];
    Holder* h = holders.get(hid);

    // generate + DF rebuild per page as before (page-granular dirty bits within holder)
    for each dirty page p of h:
        generate.dispatch_page(cmd, ring.next_slot(), p, heightmap);
        df.rebuild(ring_slot, p.bounds);

    // collect 4 ring slots + validity mask for holder
    gpu::Texture3D[4] vols;
    IVec4 valid_mask = { 0, 0, 0, 0 };
    for (int k = 0; k < 4; k++) {
        if (h.pages[k] is resident) {
            vols[k] = ring.slot_of_page(h.pages[k]);
            valid_mask.x |= (1 << k);
        } else {
            vols[k] = gpu::TEXTURE3D_INVALID;
        }
    }
    raymarch.dispatch_holder(cmd, vols, &df, h, h.pages, &h.fbo_layer0, valid_mask);
}
```

Note: M7 reintroduces **page-level** dirty bits (todo 14) — at that point this loop splits back into "regen only dirty pages, but still raymarch at holder granularity." Structure leaves room for that split.

**Cost trade-off:**
- Dispatch count: `N_dirty_pages → N_dirty_holders` (worst-case ≤ N_dirty_pages, typically ~¼ of it).
- Per-dispatch pixels: holder FBO (256×256) vs. per-page sub-rect (128×128) — 4× more pixels per dispatch, but offset by 4× fewer dispatches → net ~equal pixel work, minus dispatch/barrier overhead.
- Per-pixel work: ~1.5–2× (4 AABB tests, DF skips still dominate).

Net: small overhead for correct iso output; acceptable at POOL_SIZE=16 with MAX_REGEN_PER_FRAME=4.

## Per-Frame Data Flow (Phase 2)

```
input → pan/zoom → streaming_update:
  for each visible holder:
    ensure_holder_exists (block.retain + holder.get_or_create + 4× page slot alloc)
  lru.evict where over capacity

drain_dirty:
  drained_pages[:]           = dirty_queue.pick_nearest_n(MAX_REGEN_PER_FRAME)
  for each drained_page p:
    generate.dispatch_page (ring slot k)
    df_rebuild (ring slot k → df)
  touched_holders = unique_holders_of(drained_pages)
  for each holder h in touched_holders:
    raymarch.dispatch_holder (4 ring slots, df, h, pages, h.fbo_layer0)

composite.dispatch (alive_mask, 16 holder FBOs → backbuffer)
```

## Implementation Order (rollback-safe)

Phase 1 todos are mostly independent; however todo-4 touches `Commands` signatures at the same sites as todo-6. Ship order:

1. **todo-10** (doc + asserts) — pure additions, no risk.
2. **todo-3** (`$feature(HOT_RELOAD)`) — self-contained, one target + one function body.
3. **todo-6** (timer) — adds new `Commands` methods; no existing signatures break.
4. **todo-4** (TextureSlot registry) — touches all image/texture binds; biggest blast radius, ships last in Phase 1.
5. **todo-12** (true-iso raymarch) — Phase 2, depends on nothing from Phase 1 technically but easier to ship after todo-4 so new raymarch uses the cleaned-up bind API.

Each todo is one commit (or one series of commits with a final milestone tag).

## Rollback

Phase 1 is independent per-todo; each todo can be reverted on its own commit.

Phase 2 deletes the top-down `dispatch_into`. Rollback means reverting the raymarch.c3 + raymarch.comp.glsl pair and restoring per-page drain_dirty. The top-down path is preserved via git history (M6 Phase 2 state), not kept in-tree.
